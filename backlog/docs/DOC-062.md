## Description

The DOC-027 retrospective (2026-04-23) exposed a class of failure: outbound URLs written from memory or pattern-guessed slip past both authoring and review. Concretely, DOC-027 shipped `https://github.com/opendatadiscovery/odd-collectors/tree/main/odd-dbt` for a push-client that actually lives at `https://github.com/opendatadiscovery/odd-dbt`. The bug made it through live-site verification because the URL rendered fine — the *target* was wrong, not the *rendering*. Gate 9 (Factual Claim Provenance) was added to CLAUDE.md and `.claude/skills/{implement,review,scan}/SKILL.md` to catch new occurrences. This item is the **systemic catch** — a sweep across the entire `docs/` tree to retire any residue that predates Gate 9.

## Acceptance Criteria

### Outbound URL inventory

- [ ] Grep the entire `../documentation/docs/` tree for these URL patterns and produce a ranked list:
  - `github.com/opendatadiscovery/*` — all links to the org's repos
  - `github.com/capitalone/DataProfiler` and similar third-party org links cited as integrations (verify the upstream project still lives there)
  - `docs.opendatadiscovery.org/*` — self-links (verify target pages exist on the current `main`)
  - External docs (`docs.snowflake.com`, `dbt.com/docs/*`, `greatexpectations.io/*`, `airflow.apache.org/*`, etc.) — verify each target page resolves
- [ ] For every URL: verify the target is reachable and the anchor text / surrounding prose still describes what the target actually is

### Gate 9 classification

- [ ] For every `github.com/opendatadiscovery/*` URL in `docs/`: resolve against `navigation/architecture.md` canonical repo table and confirm:
  - Link text matches the repo's role (platform / spec / collector / push adapter / SDK / generated contracts / tooling / deployment / examples / meta)
  - Prose near the URL uses the canonical vocabulary from `navigation/architecture.md` — adapter (mapper, push or pull), collector (container of pull adapters), plugin (configured adapter in a collector), push adapter (alias: push-client)
  - URL target matches the canonical URL character-for-character (no trailing slash drift, no `/tree/main/` injection for repos that are their own thing)
- [ ] For any mismatch: fix the URL **and** add a footnote in the item's Context recording the original (wrong) URL, the file:line, and the class of error (wrong repo, wrong subpath, pattern-guessed prefix)

### Terms & Aliases reconciliation

- [ ] If a URL's surrounding prose uses a term (e.g., "Pandas profiler", "dbt adapter") that doesn't match the canonical term in `docs/main-concepts.md`, add an alias row in the same PR. Do not silently replace the term in prose without registering it.

### Guardrails shipped

- [ ] Add a new scanner definition at `scanners/docs/quality/outbound-urls.md` (in the odd-team repo) codifying this audit as repeatable — so future doc PRs that introduce new URLs are swept by `/scan` rather than by another retrospective. Scanner method: enumerate URLs by regex, resolve each against `navigation/architecture.md`, WebFetch the unresolved tail, report mismatches as findings.
- [ ] Regenerate / seed the coverage manifest for the new scanner (`state/coverage/docs-quality-outbound-urls.yaml`) with all current `docs/` files as items.

### Exit checks

- [ ] Zero `github.com/opendatadiscovery/*` URLs remain in `docs/` that can't be resolved against the architecture map
- [ ] Every URL change is in the commit footer `Sources:` block with class `Repo:` or `Integration:` citing the architecture map row
- [ ] Live-site verification at `/review` confirms each touched page renders and links resolve

## Implementation Notes

- This is a high-leverage sweep but not automated — run it once manually during implementation, encode the repeatable checks into the new `outbound-urls.md` scanner so future scans keep the property alive.
- Don't modify application code. If a URL points at a real repo but the URL is still useful (e.g., a docs page of a tool), keep it but verify reachability.
- If a target page has been moved (e.g., external docs reorganized), update the URL to the canonical new location and add an alias row if the term changed.
- If the trip reveals an upstream doc defect in another ODD repo's README (e.g., `odd-collectors/README.md` links to a stale URL), draft an `issues/{repo}/ISS-NNN.md` via `/log-issue` — we fix our own doc in this item, file the upstream fix.
