## Description

Features.md lines 129-134 list 4 activity event types: `CREATED`, `DELETED`, `UPDATED`, `ADDED (ASSIGNED)`. These don't map to reality — `ActivityEventTypeDto` has 26 enum values and **none** is named `DELETED`. Entity deletion is tracked via `DATA_ENTITY_STATUS_UPDATED`. Users trying to filter activity by the "DELETED" category see nothing.

## Acceptance Criteria

- [ ] The 4 fake categories are removed
- [ ] The documentation either (a) lists the ~19 user-visible event types matching the Activity filter UI, or (b) groups them into meaningful buckets that accurately map to real events
- [ ] The fact that entity deletion surfaces as `DATA_ENTITY_STATUS_UPDATED` is called out (it's the highest-search-value case)
- [ ] The 7 internal-only event types that are filtered out of the UI are either hidden from docs or marked as internal

## Context

- Finding: F-014 in `findings/docs-accuracy-feature-behavior/2026-04-20-batch-2.md`
- Evidence: `ActivityEventTypeDto.java` (26 values); UI `Filters.tsx` exposes ~19
