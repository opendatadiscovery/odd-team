## Description

The `/review` Step 4 (navigation consistency check) on the post-merge state of `feature/docs-active-platform-features-polish-2026-05-06` (PR #63 merged at `749fee6` on 2026-05-06; commits `63d3510` DOC-139 + `a9d115b` DOC-140) found `navigation/domains/genai.md` carrying the same factual error DOC-140 corrected on `docs/active-platform-features/genai.md` and DOC-089 corrected on `docs/configuration-and-deployment/odd-platform.md` (canonical home). Two stale lines:

- **Line 20** (Configuration block): `\`genai.url\` — external AI service URL (Java field default: empty string \`""\`; commented application.yml example: \`http://localhost:5000\`)`
- **Line 37** (Caveats block): `\`genai.url\` defaults to empty string (Java field default). Documented + flagged for upstream fix (PLT-008).`

Both claim `String url`'s Java field default is the empty string. Wrong. `String url;` in `GenAIProperties.java:10` has **no field initializer** — JVM default for reference types is `null`, not `""`. The original DOC-080 (2026-04-30) authored both lines under the older "empty string" misframing; DOC-089 (2026-05-03) corrected the canonical home at `docs/configuration-and-deployment/odd-platform.md` L1021-1025; DOC-140 (this batch) corrected the feature page at `docs/active-platform-features/genai.md` L101 and the upstream-issue draft at `issues/odd-platform/PLT-008.md`. The workspace navigation pointer was not updated alongside either correction.

Verified via Read of `GenAIProperties.java:8-12`:

```java
@ConfigurationProperties("genai")
@Data
public class GenAIProperties {
    private boolean enabled;
    private String url;
    private int requestTimeout;
}
```

Three private fields, no initializers. `String url` → `null` per JVM reference-type default. `int requestTimeout` → `0`. `boolean enabled` → `false`. The `int` and `boolean` defaults on lines 21+38 are correct (already say `0` / `false` consistently).
