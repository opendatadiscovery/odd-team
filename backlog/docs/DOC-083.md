## Description

User feedback 2026-04-30 surfaced a class of failure that the existing Quality Bar (Gates 1-9) did not catch: **content of a recognizable type authored on a feature page when a canonical home for that type exists elsewhere in the doc tree, but is empty or sparse.** The trigger case:

- `https://docs.opendatadiscovery.org/master-data-management/lookup-tables#api-reference` carries ~1300 words of detailed HTTP endpoint tables (16 endpoints across 4 groups: Table CRUD, Column CRUD, Row CRUD, Search). Operation IDs, paths, methods, response schemas — all there.
- `https://docs.opendatadiscovery.org/developer-guides/api-reference` carries only a "use Swagger UI" pointer. No endpoints, no operation IDs, no schemas. It is the dedicated page that should be the canonical home for HTTP endpoint documentation, but it sits empty.

This is **not** Gate 1 duplication (the content isn't duplicated; it exists in only one place). It is a **content-type homing failure**: API-reference content lives on a feature page when the canonical home for that type is the dedicated page.

The 2026-04-30 bigbatch shipped four MORE feature pages with the same drift class:

- `docs/configuration-and-deployment/odd-platform.md` `### API surface` for data-collaboration (DOC-034) — 7 routes across 3 tables (Outbound to Slack, Per-entity threads/history, Inbound webhook).
- `docs/directory.md` `## API surface` (DOC-047) — 4 endpoints (Directory hierarchy levels 1-4).
- `docs/integrations/integration-wizard.md` `## API surface` (DOC-050) — 2 endpoints (`getIntegrationPreviews`, `getIntegration`).
- (existing) `docs/lookup-tables.md` `## API reference` — 16 endpoints (the trigger case).

In every case the work shipped reads well in isolation, but the global picture is: **API-reference content is scattered across feature pages, the dedicated `developer-guides/api-reference` is empty, and a developer asking "where do I find every HTTP endpoint?" has no canonical answer.**

The Quality Bar was extended on 2026-04-30 with **Cornerstone 5 (One canonical home per content type)** and **Gate 10 (Content type homing)** to prevent recurrence. This item is the systemic refactor that brings the existing tree up to the new bar.
