## Description

Two Spring-level config keys shipped with the platform have security and operational implications and are not documented:

- `spring.session.timeout` — default `-1` (infinite session). Security-sensitive because sessions never expire unless explicitly configured.
- `spring.codec.max-in-memory-size` — default `20MB`. Controls the maximum request body size, which effectively caps file uploads and large API payloads.

## Acceptance Criteria

- [ ] `spring.session.timeout` is documented in the session-provider section with its default and a note explaining that `-1` means "never expire" and how to set a finite timeout
- [ ] `spring.codec.max-in-memory-size` is documented near the attachment/upload configuration with its default and units
- [ ] Both keys include env-var equivalents
- [ ] A brief security note covers the implication of the infinite-session default
