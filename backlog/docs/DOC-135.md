## Description

Editorial audit on `/review batch:feature/docs-bigbatch-2026-05-06` surfaced a **parallel-surfaces-with-drift** defect: the same Collector category-vs-implementation conflation that DOC-099 Phase A fixed on `Architecture.md:11` lives unfixed on `trylocally.md:10`.

### The defect

`docs/configuration-and-deployment/trylocally.md:10` (in the "This environment consists of:" bullet list):

> * ODD Collector – a lightweight service which gathers metadata from all your data sources

This phrasing repeats the conflation that DOC-099 Phase A diagnosed: it presents "ODD Collector" as a single thing — *the* lightweight service — when in fact:

- **Collector** is a *category* (a container of pull adapters plus the runtime around them, per `main-concepts.md` Terms & Aliases).
- **odd-collector** is one specific implementation in that category (alongside `odd-collector-aws`, `odd-collector-azure`, `odd-collector-gcp`, `odd-collector-profiler`).

The demo's `docker/demo.yaml` bundles `odd-collector` (the canonical generic implementation), not "the" Collector. The current sentence implies the latter — same conflation Architecture.md L11 had before DOC-099 Phase A fixed it.

### Why this slipped past DOC-099

DOC-099 Phase A's scope was Architecture.md L11 specifically. The audit didn't sweep parallel surfaces for the same conflation pattern. trylocally.md and Architecture.md describe the same concept (the Collector component) for different audiences (demo operator vs architectural overview), so the drift is exactly the "parallel surfaces with drift" failure shape from `playbooks/doc-product-editorial-read.md`.

### Reader-experience cost

A new reader on the demo page forms a mental model where "ODD Collector" is *the* collector — a single deployable. They then read main-concepts.md or Architecture.md and find Collector is a category with five implementations. The two surfaces send incompatible conceptual signals. Per the world-class doc bar (every reader should land on the same canonical taxonomy regardless of which page they entered through), this is a coherence finding.

## Failure shape

**Parallel surfaces with drift** (per `playbooks/doc-product-editorial-read.md` failure-shape table). The canonical Collector-as-category framing is on `main-concepts.md` (post-DOC-115) and `Architecture.md` (post-DOC-099 Phase A); `trylocally.md:10` retains the pre-DOC-099 single-thing framing.
