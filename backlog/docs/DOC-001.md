## Description

A full Azure AD OAuth2 provider is implemented in `odd-platform-api/src/main/resources/application.yml` (lines 128-156) but is completely absent from the OAuth2/OIDC documentation. The docs cover Cognito, GitHub, Google, OKTA, and Keycloak — Azure AD is one of the most widely used enterprise identity providers and its omission is a critical coverage gap.

The implementation supports both multi-tenant and single-tenant deployments and exposes 15+ config keys including `azure-tenant-id`, `authorization-uri`, `token-uri`, `user-info-uri`, `admin-attribute`, `admin-principals`, and `admin-groups`.

## Acceptance Criteria

- [ ] `oauth2-oidc.md` includes a new Azure AD section alongside the existing providers
- [ ] Multi-tenant configuration is documented with full example (client-id, client-secret, scopes, user-name-attribute)
- [ ] Single-tenant configuration is documented with `azure-tenant-id` and the tenant-specific issuer-uri/authorization-uri/token-uri/user-info-uri
- [ ] Admin attribute mechanism is explained (`admin-attribute`, `admin-principals`, `admin-groups`) with at least one worked example
- [ ] Required Azure AD app registration steps are listed (redirect URI, API permissions — openid, offline_access, graph)
- [ ] Env var equivalents are provided for each property, matching the format used in existing provider sections
