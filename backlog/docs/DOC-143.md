## Description

Carve out the **Manual Object Tagging** content from `docs/Features.md` (L142-150) into a dedicated `docs/data-discovery/tagging.md` page. Today's section is thin (one paragraph + a sub-heading + the `tagging.gif` reference) but tagging is operator-meaningful as a Discovery surface — tags drive faceted search (already documented as the Tag facet on the new Search page from DOC-142), and tag curation is an operator workflow distinct from search and from directory navigation.

The new page will need **enrichment beyond what's on Features.md today** to justify its own home: cover the platform-side Tag entity, the per-tag detail view, the Important-tag flag (rendered visually distinct), namespace-scoped tags, and the relationship to the Tags Management surface (`/management/tags` — already linked from `management.md`). Operator workflow guidance for "how to author a stable tag vocabulary" is implicit in Management today; making the user-side tagging surface explicit here closes the loop.

Sub-task of the umbrella **DOC-141**.
