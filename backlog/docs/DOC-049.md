## Description

The platform supports operator-configurable additional navigation links (`LinksController.java` → `GET /api/links`, backed by `AdditionalLinkProperties`). Administrators can add links (title + URL) that surface in the platform UI — useful for pointing users at internal wikis, runbooks, or dashboards. Zero documentation exists.

## Acceptance Criteria

- [ ] The platform configuration page documents the additional-links config property
- [ ] Config structure is shown (array of link objects with title + URL, per `AdditionalLinkProperties`)
- [ ] A YAML example and env-var equivalent are provided
- [ ] Where the links render in the UI is described

**Caveats surfaced (shipped in the same commit):**
- **In-scope, shipped:** every signed-in user can read the configured links via `GET /api/links` and via the App Info menu — operators must not embed secrets in link URLs. Documented as a `hint style="info"` admonition under the new subsection.
- **Out-of-scope, not logged as upstream issue:** the platform does not validate that `url` is a well-formed absolute URL or that `title` is non-empty. Severity is low (operator-supplied config, immediate UI feedback when broken — clicking a malformed link does nothing useful). No PLT issue drafted.
