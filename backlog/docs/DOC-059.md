## Description

DOC-006 corrected the `SESSION_PROVIDER` example value from `IN_MEMORY` to `INTERNAL_POSTGRESQL` — a 1-line fix. A consumer-read re-audit of `SessionConfiguration.java` and the housekeeping job surfaced operator-visible behaviors for each of the three providers that are not documented anywhere.

Findings:

- **`IN_MEMORY`** — `ReactiveMapSessionRepository` wraps a raw `ConcurrentHashMap`. Sessions are only evicted by Spring Session's expiry check; there is no secondary eviction policy. A long-running process with many short-lived sessions can accumulate map entries until the Spring Session TTL expires them. Under aggressive load this is an OOM risk. Not documented.

- **`INTERNAL_POSTGRESQL`** — `PostgreSQLSessionHousekeepingJobHandler.deleteExpiredSessions()` is annotated `@Scheduled(fixedRate = 1, timeUnit = TimeUnit.HOURS)`. Expired sessions therefore sit in `SPRING_SESSION` / `SPRING_SESSION_ATTRIBUTES` for up to **one hour** past their TTL before the job cleans them. The cadence is not configurable. Not documented — an operator sizing the Postgres pool / disk will under-estimate session-table growth.

- **`REDIS`** — the `@EnableRedisWebSession` bean has no local connection wiring; operators must supply `spring.data.redis.*` (or `spring.redis.*` on older Spring Boot) to configure host, port, credentials, TLS, and pool sizing. None of these keys are documented in `odd-platform.md`. Without them, the platform will fail to start against a non-default Redis deployment.

## Acceptance Criteria

- [ ] Session-provider section in `odd-platform.md` documents each of the three providers with a "Characteristics / caveats" subsection
- [ ] `IN_MEMORY`: warns that sessions live in JVM heap, lost on restart, no multi-instance support; notes OOM risk for long-running high-cardinality-session deployments
- [ ] `INTERNAL_POSTGRESQL`: documents that session rows are cleaned by an hourly housekeeping job (hardcoded, not configurable); recommends sizing `SPRING_SESSION` table for up to 1 hour of post-expiry accumulation
- [ ] `REDIS`: documents the required `spring.data.redis.*` keys an operator must set (host, port, credentials, ssl, timeout), with a working example and a note that TLS / pool defaults come from Spring
- [ ] All three subsections include env-var equivalents (if applicable) and cross-reference each other
