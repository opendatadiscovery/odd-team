## Description

Editorial audit on `/review batch:feature/docs-bigbatch-2026-05-06` surfaced a **Gate-3 caveat-as-admonition** defect: a known limitation operators will hit in production is presently embedded in narrative prose rather than rendered as a `{% hint style="warning" %}` block.

### The caveat in question

`docs/Features.md:424` (inside the Data Collaboration section's introductory paragraph):

> The feature is **disabled by default** (`datacollaboration.enabled=false`). When disabled, every `/api/datacollaboration/...` route plus the `/api/slack/events` webhook return `404 Not Found` — the controllers are gated by `@ConditionalOnDataCollaboration`. **The Discussions tab itself is currently rendered regardless of the flag (only hidden when the entity status is deleted), so on a deployment that has not configured Data Collaboration the tab is visible but its content fails to load — operators who do not plan to enable the feature should be aware of this until the platform gates tab visibility on the flag.**

The bolded sentence is the caveat. It documents an operationally relevant gap: an operator who deploys the platform without configuring Data Collaboration sees a `Discussions` tab on every entity but clicking it fails to load anything (the `/api/datacollaboration/...` routes return 404). This is the kind of nuance a Principal Full-Stack maintainer would expect to surface as a warning block, not as the third sentence of a paragraph.

### Why this is a Gate-3 defect class

Per `pillars/documentation/gates.md` Gate 3 (Caveats captured): "Every caveat the consumer-read audit surfaced is in the doc as an admonition block, not buried in prose." The L424 caveat satisfies the **captured** half of the requirement (it's in the doc) but fails the **as an admonition block** half (it's prose).

Adjacent caveats on the same page — the Distribution-anomaly halt unenforced (L228-236), the Manual reopen guard (L261-263), the Manually-resolved alerts deletion bug (L271-277) — are all rendered as `{% hint style="warning" %}` or `{% hint style="info" %}` blocks. The L424 caveat is the outlier.

### Reader-experience cost

A reader skimming the Data Collaboration section for "what do I need to know before deploying this" can miss a buried-in-prose caveat. A warning admonition is visually distinct, clearly marked as a known limitation, and survives skim-reads. The current placement asks the operator to read every word of the introduction paragraph to catch the limitation.

## Failure shape

**Gate 3 — Caveat captured but not as admonition** (per `pillars/documentation/gates.md` Gate 3 + `playbooks/doc-product-editorial-read.md` "dead admonitions / buried admonitions" failure shape).
