## Description

Create a new **Data Glossary** pillar under `## Features` in SUMMARY, mirroring the DOC-131 pattern. Today the Business Glossary feature is fragmented across two H2 sections on Features.md (`## Dictionary terms` L184-201 + `## Associating Terms with Data Entities through Descriptive Information` L259-307) plus a row in Main Concepts → Terms & Aliases (L77 — `Business Glossary (feature) | Glossary, Terms | …`). The two H2s are the same feature at two scales — terms as catalog entities (Dictionary tab) and terms as descriptive metadata on data entities (inline mentions). One canonical home is missing.

The pillar collects the full Business Glossary surface: terms as entities (create / edit / namespace-scope / ownership / RBAC), the Dictionary tab UI walkthrough, term-to-term linking (description-text mentions vs direct term-to-term — already a sub-section on Features.md L191-201), the descriptive-information walkthrough (Wikipedia-About-style term descriptions, term-as-link-format, reverse search, term-to-column linking — Features.md L259-307), and the API surface (cross-link to API Reference → Glossary).

Sub-task of the umbrella **DOC-141**.
