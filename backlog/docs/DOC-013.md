## Description

`odd-collectors` supports a `secrets_backend` block in `collector_config.yaml` that lets operators store the collector token and per-plugin credentials in AWS Systems Manager Parameter Store instead of in plaintext YAML. This is a production-critical security feature — in its absence, credentials must be checked into config files. The feature is currently undocumented; the only explanation lives in a comment block at the top of `collector_config.yaml`.

Feature surface (from `odd-collector/collector_config.yaml` lines 1-34):

- `secrets_backend.provider: AWSSystemsManagerParameterStore`
- `secrets_backend.region_name`
- `secrets_backend.collector_settings_parameter_name`
- `secrets_backend.collector_plugins_prefix`
- Secrets override file config when both are present

## Re-Audit (2026-04-23)

Flipped from `done` to `review-ready` after consumer-read re-audit under the new eight-gate Quality Bar (Phase C of the pipeline-hardening effort).

**In-scope amendment required and shipped on `feature/doc-013-secrets-backend-caveats` (documentation), commit `ddb0aff`.** The Gate 5 unset-parameter audit on the boto3 SSM client builder (`ssm_parameter_store.py:31`) surfaced four operator-visible caveats that the original DOC-013 page did not document — matching the DOC-008 `.region(...)` class of finding:

- `ssm_parameter_store.py:144` — `get_parameters_by_path(..., Recursive=True)` is called **once**, not in a paginator loop. AWS default `MaxResults` for this call is **10**. A collector with >10 plugin parameters under `collector_plugins_prefix` silently starts with only the first 10; the rest are dropped with no warning. This is a data-loss-shaped trap for anyone scaling a single collector beyond a handful of plugins.
- `ssm_parameter_store.py:31` — `boto3.client("ssm", region_name=...)` is constructed without `endpoint_url`. VPC interface endpoints, LocalStack targets, and any private DNS override cannot be configured via `secrets_backend.*`.
- `ssm_parameter_store.py:31` — no `botocore.config.Config` is passed; SDK default connect timeout, read timeout, and retry mode apply. A slow or partitioned SSM endpoint blocks startup for the full default timeout.
- `ssm_parameter_store.py:165,176` — `safe_load(p["Value"])` with no surrounding try/except. A malformed YAML document in any plugin parameter aborts startup with a `LoadConfigError`-wrapped yaml parse error that does not identify the offending parameter.

Also corrected the Step 3 region-resolution wording: the collector process does start with `region_name=None`, but the first SSM call raises `botocore.exceptions.NoRegionError` and the collector exits before ingestion begins. "Fails to start" was operationally-true but technically imprecise.

**Classification**: in-scope for DOC-013 (remit: "document the secrets backend feature completely"; defaults and failure modes are part of complete coverage under Quality Bar #3 + #5).

**Platform-side follow-up filed**: the `get_parameters_by_path` pagination bug is a collector-SDK defect, not just a doc gap. The doc-side caveat ships in DOC-013 (Known limitations admonition); the SDK fix is tracked on disk as a paste-ready GitHub issue draft. On-disk issue draft: `issues/odd-collectors/COL-001.md` (status: `draft`; paste into `github.com/opendatadiscovery/odd-collectors/issues/new` when ready to file). Remove the doc caveat once a release with pagination lands.
