## Description

Create a new **Data Quality** pillar under `## Features` in SUMMARY, mirroring the DOC-145 (Data Lineage) and DOC-146 (Data Glossary) pattern. Today the DQ feature surface is fragmented across four H2 sections on Features.md:

- `## Data Quality Test Results Import` (L115-124) — the integrations that push test results into the platform: Great Expectations, dbt tests, `odd-collector-profiler` (uses Capital One DataProfiler), and the custom-framework `POST /ingestion/entities/datasets/stats` endpoint.
- `## Dataset Quality Statuses (SLA)` (L211-219) — the operator-set Minor / Major / Critical status on test results, with the per-dataset SLA URL `/api/datasets/{data_entity_id}/sla` for BI-report import.
- `## Data Quality Dashboard` (L373-397) — the catalog-wide DQ dashboard at `/data-quality` (already screenshot-captured 2026-05-07 batch as `docs/.gitbook/img/data-quality.png` and described inline at Features.md L122-124). The six anomaly-class metrics (Assertion Tests / Custom Values / Freshness / Schema Changes / Unknown / Volume).
- `## Data Quality Dashboard Filtering` (L398-413) — the filter dimensions (Namespace / Datasource / Owner / Title / Tag) for tables-side and tests-side filtering separately, with the `AND`-only conjunction caveat.

All four are read-side surfaces of the same Data Quality feature. Cornerstone 5 binds — one canonical home; this pillar consolidates them.

Per the user's request 2026-05-07 to add Data Quality alongside Data Lineage and Data Glossary as a dedicated pillar.

Sub-task of the umbrella **DOC-141**.
