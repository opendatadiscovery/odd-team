## Description

DOC-042 shipped the `odd-collector-azure` integrations page with the full inventory (4 adapters) and **spotlight per-adapter config tables for 2 representative adapters** (`powerbi`, `blob_storage`). The remaining **2 adapters** are listed in the inventory but do not yet have inline per-field config tables.

This item completes the per-adapter user-facing reference: every type literal in `odd_collector_azure.domain.plugin.PLUGIN_FACTORY` gets a sub-section with a field-by-field table.

The 2 adapters to cover:

`azure_sql`, `azure_data_factory`.

## Implementation Notes

- `azure_sql` does NOT inherit from `AzurePlugin` (it inherits directly from `Plugin`) — its auth surface is `username` + `password` + `server` + `database` + `port`. Different from the `client_id` / `client_secret` / `domain` pattern PowerBI uses.
- `azure_sql` defaults: `encrypt="yes"`, `trust_server_certificate="no"`, `connection_timeout="30"` (note: all string-typed in the Pydantic model — that is suspicious, may warrant verification against the consuming code).
- `azure_data_factory` requires environment-variable auth via `DefaultAzureCredential` — re-state this caveat explicitly per AC, not just on the common section.
