## Description

`oauth2-oidc.md` line 231, under the OKTA provider's env var example, sets `AUTH_OAUTH2_CLIENT_OKTA_PROVIDER=google`. Users who copy-paste this will configure OKTA-shaped auth with `provider=google`, which mismatches the client-id under the `okta.*` key and causes authentication failures. The YAML tab (line 215) correctly uses `provider: okta` — only the env-var tab has the bug, which is a classic copy-paste from the Google section above.

## Acceptance Criteria
