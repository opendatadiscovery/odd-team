## Description

17 `"mention"` shortcut links remain across 8 files in `docs/configuration-and-deployment/`. These are the GitBook editor-native internal-reference shortcut (`[text](target.md "mention")`) hand-authored in git. Per the project authoring rules (`CLAUDE.md` → "Documentation Authoring Rules"), this shortcut is **unreliable when authored outside the GitBook editor** and can silently fall back to raw `github.com/opendatadiscovery/documentation/blob/main/...` URLs which then get cached on the live site. Every instance must be rewritten as a plain markdown link.

The 2026-04-22 S2S incident is the canonical precedent: a `"mention"` link in an index page fell back to GitHub after a separate-PR merge, and the fallback URL stayed cached. Rewriting proactively prevents the same failure class.

## Context

- Some links (e.g., index-page `"mention"` lists) may currently render correctly on the live site because GitBook resolves them when the target file exists at a stable path. The risk is intermittent: any file rename, PR-merge ordering change, or GitBook re-index can flip the resolution to the GitHub fallback. The authoring rule is "never hand-author" precisely because the failure is not deterministic.
- Three of these files (`odd-platform.md`, `enable-security/README.md`, `permissions.md`) already have `in-progress`/`pending` work items touching them (DOC-004, DOC-005..011, DOC-018, DOC-019, DOC-034, DOC-037, DOC-045, DOC-049). ## Implementation Notes

- Pick reader-facing titles from the target page's H1 (e.g., `roles.md` → `# User Roles` → link text "User Roles"), not the filename.
- For directory links (`enable-security/`, `authentication/`, `authorization/`), link to the directory's `README.md` by its title — e.g., `[Authentication](authentication/README.md)`.
- The `owners.md:9` and `roles.md:19,21` occurrences are inline mid-sentence — preserve sentence grammar ("This process is described in the [User-Owner Association](user-owner-association.md) section.").
- After editing, run `rg '"mention"' docs/` as the final check — zero matches is the gate.
- Batch well with any other `configuration-and-deployment/` work (e.g., as part of `feature/docs-quality-authoring-hygiene` or the next `enable-security` batch).
