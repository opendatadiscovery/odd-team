## Description

The collector SDK's `CollectorConfig` class (`odd-collector-sdk/odd_collector_sdk/domain/collector_config.py` lines 13-27) defines 8 config fields, but the documentation (`build-and-run-odd-collectors.md` lines 84-92) only covers 4. Missing from the docs:

- `connection_timeout_seconds` (default: 300)
- `chunk_size` (default: 250)
- `misfire_grace_time` (default: None)
- `max_instances` (default: 1)
- `verify_ssl` (default: True)

Users running collectors against self-signed-cert platforms can't discover `verify_ssl=False`. Users with large metadata sets can't find `chunk_size`. Users with slow sources don't know about `connection_timeout_seconds`.

## Acceptance Criteria

- [ ] The collector configuration reference in `build-and-run-odd-collectors.md` documents all 8 `CollectorConfig` fields (or is expanded into a dedicated reference page)
- [ ] Each field has: type, default, and a short description of when to change it
- [ ] `verify_ssl` documentation explicitly mentions the self-signed-cert use case
- [ ] `chunk_size` documentation explains the trade-off (memory vs. throughput) and notes where to tune it for large datasets
- [ ] `connection_timeout_seconds` documentation notes that it applies to HTTP calls to the platform (not to plugin sources)
