## Description

Editorial audit on 2026-05-03 surfaced three residual link-quality defects on `docs/Features.md` that DOC-094…DOC-096 + DOC-101 (review-ready in this batch) did not address.

### Site A — line 80: trailing space inside link text + magic-link to Java source

```
Let's explore the [technical details ](https://github.com/opendatadiscovery/odd-platform/blob/main/odd-platform-api/src/main/java/org/opendatadiscovery/oddplatform/repository/util/FTSConstants.java).
```

Two defects:
1. **Trailing space** inside the link text `[technical details ]` — same defect class DOC-095 fixed for `[Activity feature ]` on line 467. GitBook may render the trailing-space underline cosmetically wrong.
2. **Magic link**: the link text says "technical details" but the target is a single Java file (`FTSConstants.java`). The reader expects "technical details" to be a documentation page or a richer overview; instead they land on a code constants file. Reader-flow defect.

### Site B — line 772: commit-hash-pinned outbound URL

```
For customization refer to the [_**application.yml**_](https://github.com/opendatadiscovery/odd-platform/blob/4572bba1fcfbee0c62183804e3c692815fa3e955/odd-platform-api/src/main/resources/application.yml#L195) file:
```

The URL pins `application.yml` at commit `4572bba1...` and line `L195`. Two defects:
1. **Commit hash pin**: as the file evolves, line 195 will eventually point at unrelated content; the doc claim "this is the attachment storage section" silently rots.
2. **Cross-pillar concern**: this links into platform-source code, but the doc tree has a canonical home for platform configuration: `configuration-and-deployment/odd-platform.md#attachment-storage-configuration`. The cross-link should target the doc page, not the source code.

### Site C — line 814: untargeted `application.yml` link

```
ODD users have the flexibility to [adjust this period](https://github.com/opendatadiscovery/odd-platform/blob/main/odd-platform-api/src/main/resources/application.yml) to better suit their needs
```

Links to the entire application.yml at HEAD with no anchor. Reader has to scroll to find the relevant `odd.data-entity-stale-period` section. The doc tree has the canonical home: `configuration-and-deployment/odd-platform.md` line 778's `odd.data-entity-stale-period` bullet. The cross-link should target the doc anchor, not the YAML file at HEAD.

## Failure shape

**Cross-link mismatch** (Site A magic link; Sites B and C wrong target — points at source rather than doc) + **outdated baked-in assumption** (Site B commit-hash pin will rot) (per `playbooks/doc-product-editorial-read.md` failure-shape table).

## Acceptance Criteria

- [ ] **Site A** (line 80): remove trailing space inside link text (`[technical details ]` → `[technical details]`). Decide on link target — either keep the magic-link to FTSConstants.java but rephrase the link text (e.g., "explore the [FTS constants source](url)") OR drop the link entirely and integrate the technical details inline.
- [ ] **Site B** (line 772): replace the commit-hash-pinned application.yml URL with a relative cross-link to the canonical doc home `configuration-and-deployment/odd-platform.md#attachment-storage-configuration`. The link text "**application.yml**" should be replaced with something more reader-relevant like "Attachment Storage Configuration" since the new target is a doc page, not the YAML file.
- [ ] **Site C** (line 814): replace the untargeted application.yml URL with `configuration-and-deployment/odd-platform.md#detecting-stale-metadata` (the canonical home for `odd.data-entity-stale-period`). Keep the link text "adjust this period" — the cross-link target now matches the action.
- [ ] **Sweep**: `grep -rEn '\[.* \]\(' docs/` — list every markdown link in the doc tree where the link text ends in a space inside the closing bracket. Fix each. (Same regex sweep should be added to `playbooks/duplication-sweep.md` step 4 as a recurring sweep.)
- [ ] **Sweep**: `grep -rEn 'github\.com/.*?/blob/[a-f0-9]{40}/' docs/` — list every outbound link pinned to a commit hash. Each is a candidate for either re-pin to a tag, switch to `/blob/main/`, or replace with a relative cross-link to the doc canonical home. Fix where appropriate; document where commit-pin is intentional.
- [ ] Live-site verification (Gate 8 / `/review`): WebFetch `https://docs.opendatadiscovery.org/features` and confirm the three replacements render correctly.
- [ ] Sources footer: `Sources: none (cross-link cleanup, no factual claim)`.
