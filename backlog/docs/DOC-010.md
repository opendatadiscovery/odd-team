## Description

Three platform-level `odd.*` config keys are entirely undocumented:

- `odd.tenant-id` — multi-tenancy identifier, no default (line 210)
- `odd.data-entity-stale-period` — days before an entity is marked stale if not re-ingested (default: 7, line 211)
- `odd.activity.partition-period` — PostgreSQL activity-feed partition period in days (default: 30, line 213)

`data-entity-stale-period` particularly matters for users running collectors on schedules longer than 7 days — they'll see entities marked stale unexpectedly. `partition-period` matters for high-volume deployments where the default partition granularity is wrong.

## Acceptance Criteria

- [ ] All three keys are documented with defaults, types, and env-var equivalents
- [ ] `data-entity-stale-period` explanation includes what "stale" means in the UI/API (how it surfaces to users)
- [ ] `tenant-id` documentation states clearly that it's intended for multi-tenant deployments and has no default
- [ ] `activity.partition-period` is grouped with activity-feed settings (or placed in an advanced/operational subsection) and explains when an operator would tune it
