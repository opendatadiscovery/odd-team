## Description

`application.yml` (line 37) ships with `auth.login-form-credentials: admin:admin,root:root` and a comment marking these as "dev/demo purposes only". The LOGIN_FORM auth documentation does not mention this property and does not warn users that default credentials are baked into the config. Users enabling login-form auth without overriding the property are running with default credentials.

## Acceptance Criteria

- [ ] `login-form.md` documents the `auth.login-form-credentials` property format (`user1:pass1,user2:pass2,...`)
- [ ] A prominent warning states that the shipped config contains default credentials (`admin:admin`, `root:root`) and that these MUST be changed before any non-local deployment
- [ ] The warning explains how to override: set the property explicitly (YAML or env var) or switch to OAuth2/LDAP
- [ ] The env-var equivalent (`AUTH_LOGIN_FORM_CREDENTIALS`) is shown

## Context

- Finding: F-053 in `findings/docs-accuracy-config-options/2026-04-21-platform.md`
- Evidence: `application.yml` line 37, inline comment "For dev/demo purposes only"
- Severity in scanner: medium (security)

## Implementation Notes

Related bug: the shipped defaults are a product decision (convenient for local dev) that trades off security. If removing the defaults is considered, that's a code change in `application.yml` — log as a separate GitHub Issue in `opendatadiscovery/odd-platform` rather than doing it here. This work item is strictly a doc warning.
