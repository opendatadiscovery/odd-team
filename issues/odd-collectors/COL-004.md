## What

The generic `odd-collector` package's `config_examples/` directory ships one reference YAML per supported adapter â€” except for two adapters that are present in `PLUGIN_FACTORY` and shipped, but have **no** corresponding example file:

- `mode` â€” `ModePlugin` is registered (`PLUGIN_FACTORY` line 381), no `config_examples/mode.yaml`.
- `opensearch` â€” `OpensearchPlugin` is registered (`PLUGIN_FACTORY` line 390), no `config_examples/opensearch.yaml`.

Operators evaluating these adapters have to read the Pydantic model in `plugin.py` to figure out the field shape â€” there is no canonical YAML to copy from.

## Where

- `odd-collector/odd_collector/domain/plugin.py:279-285` (`ModePlugin`)
- `odd-collector/odd_collector/domain/plugin.py:337-346` (`OpensearchPlugin`)
- `odd-collector/config_examples/` directory lacks `mode.yaml` and `opensearch.yaml`. Listing of the directory shows files for every other PLUGIN_FACTORY entry.

## Why it matters

- Every other adapter in `odd-collector` has a `config_examples/{type}.yaml` reference YAML. New operators landing on the repo expect that pattern â€” its absence for `mode` and `opensearch` looks like the adapters are not fully supported, when in fact they are.
- For the documentation pipeline, missing examples mean the doc author has to hand-craft a YAML from the Pydantic model, which adds drift risk over time as the model evolves.
- Severity is **low** â€” the adapters work; the gap is a discoverability and DX issue.

## Suggested fix

Add the two missing files. A minimal-but-complete YAML for each, matching the pattern of the existing examples (single adapter under a `plugins:` list, with the typical `platform_host_url` / `default_pulling_interval` / `token` envelope), is straightforward to author:

```yaml
# config_examples/mode.yaml
platform_host_url: http://localhost:8080
default_pulling_interval: 10
token: ""
plugins:
  - type: mode
    name: mode_adapter
    host: https://app.mode.com
    account: my_workspace
    data_source: my_warehouse
    token: <mode_api_token>
    password: <mode_password>  # legacy auth, optional if using token
```

```yaml
# config_examples/opensearch.yaml
platform_host_url: http://localhost:8080
default_pulling_interval: 10
token: ""
plugins:
  - type: opensearch
    name: opensearch_adapter
    host: https://search-mydomain.us-east-1.es.amazonaws.com
    port: 443
    use_ssl: true
    username: opensearch_admin
    password: <opensearch_password>
    verify_certs: true
```
