## Description

Features.md lines 83-88 claim the platform "helps track and compare your experiments" and supports logging "the most successful experiments." None of this exists in code. `ML_EXPERIMENT` is a `DataEntityType(6)` that rolls up under `DATA_ENTITY_GROUP` — a grouping primitive only. There is no dedicated experiment UI, no comparison view, no "success" or "logging" machinery. Users expecting MLflow-like functionality are being sold something that isn't there.

## Acceptance Criteria

- [ ] The ML Experiment section is rewritten to describe what actually exists: ML experiments are data entity groups that organize related ML entities (models, datasets, training jobs) in one logical container
- [ ] Claims about "compare", "track", and "log the most successful" are removed
- [ ] A note clarifies that for experiment tracking itself, operators should use MLflow or equivalent and push metadata to ODD rather than expect ODD to replace the tracker
- [ ] The section is moved or linked near the data-entity-groups description (they're the same mechanism)

## Context
