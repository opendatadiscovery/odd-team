## Description

The "Enable Data Collaboration" section describes the feature at a high level (Slack threads etc.) but omits the dedicated API endpoints:

- `/api/datacollaboration/providers/slack/channels`
- `/api/datacollaboration/providers/slack/messages`
- `/api/dataentities/{id}/messages/*`

Additionally, the Slack Events webhook URL pattern shown in the manifest template (setup instructions) may not match the actual endpoint served by the code (`/api/slack/events`). This needs verification.

## Acceptance Criteria

- [ ] The data collaboration section references the key API endpoints (at least the three listed above)
- [ ] The Slack Events webhook URL in the manifest template matches the endpoint handled by code — verified against `odd-platform-api` routes
- [ ] If a mismatch is found in the manifest template, it is corrected in the docs
- [ ] A note mentions `message-partition-period` configuration (or its current binding) if documented separately
