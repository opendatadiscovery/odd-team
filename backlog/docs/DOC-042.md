## Description

`collector.md` is a 1-line empty page. `Adapters.md` (line 7) tells users "Adapters will be deprecated soon... we recommend you to install the Collector." and implicitly routes readers toward collector info — but the only destination is this empty page. Users following the deprecation notice hit a wall.

## Acceptance Criteria

- [ ] **Option A (populate)**: `collector.md` becomes a real collector overview page — what a collector is, how it relates to adapters, how to pick between sub-collectors (generic/AWS/GCP/Azure), link to `build-and-run-odd-collectors.md` and `configuration-and-deployment/` for deeper content. Source material: `odd-collectors` READMEs and existing nav
- [ ] **Option B (remove)**: page deleted, `Adapters.md` link in SUMMARY updated to link directly to `build-and-run-odd-collectors.md` or `configuration-and-deployment/`
- [ ] `Adapters.md` links are updated either way so they point to a non-empty target
- [ ] The decision is justified in the commit
