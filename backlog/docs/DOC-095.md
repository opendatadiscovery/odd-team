## Description

Editorial audit on 2026-05-03 (during `/review batch:feature/docs-features-md-spotcheck`) surfaced a broken cross-link on `docs/Features.md:467`:

```
Moreover, the [Activity feature ](https://docs.opendatadiscovery.org/\~/changes/pcY8r1Ougw5Sv8zDaz8G/features#activity-feed-for-monitoring-changes)serves as a tracking system for different changes...
```

The URL `https://docs.opendatadiscovery.org/\~/changes/pcY8r1Ougw5Sv8zDaz8G/features#...` is a **GitBook preview-mode URL**. The `/~/changes/{hash}/` path is GitBook's draft-preview surface — accessible only to GitBook editors viewing a specific in-progress change set. On the published site this URL returns either a 404 or a redirect to an unrelated page; for any external reader it is a dead link.

The fix is to replace the absolute preview URL with a relative intra-doc anchor: `[Activity feature](Features.md#activity-feed-for-monitoring-changes)` (or just `[Activity feature](#activity-feed-for-monitoring-changes)` since this is a self-page reference). Per `pillars/documentation/authoring.md` "GitBook authoring" section, intra-repo cross-links must be relative paths — the same rule DOC-092 enforced for the Medium-tracking URLs in the `## Alerting` section.

Side issue (polish): the link text `[Activity feature ]` has a **trailing space** before the closing bracket, which produces an off-by-one underline in some markdown renderers. Fix while editing.

## Why this slipped past prior batches

The DOC-092 sweep `grep -rn 'source=post' docs/` matched only the Medium-article URL pattern (the four sites on Features.md DOC-092 fixed). The GitBook preview pattern (`/~/changes/`) is a different defect class with the same root cause (an absolute URL that should be a relative anchor) and was not in the regex set.

Worth extending the cross-link sweep regex to catch all `https?://docs\.opendatadiscovery\.org/.*` patterns in the doc tree — every such URL is a candidate for relativisation. Log as a follow-on if this fix surfaces a third pattern.

Editorial-audit case-law for `playbooks/doc-product-editorial-read.md` — failure shape: Cross-link mismatch (link text promises one thing, target either fails or delivers another). Plus a Gate-1-bidirectional-sweep extension opportunity.

## Context

- **Failure shape**: Cross-link mismatch (per `playbooks/doc-product-editorial-read.md` failure-shape table).
- **Why high (not medium)**: a reader following the "Activity feature" link from the Schema Diff section gets a 404 or unrelated page on the live site. The link is functionally broken, not just stylistically wrong.
- **Why not critical**: the same content the link targets is reachable via the in-page TOC at the top of Features.md or via search; reader can recover. But a careful operator who depends on the doc product reads cross-links as authoritative.
- **Recurring pattern**: GitBook editor sometimes inserts preview-mode URLs when an editor pastes a link from their open preview tab. Worth a Gate-1 sweep extension — see ACs above.
