## Description

Resolved alerts are silently deleted after 30 days by the `AlertHousekeepingJob`. The alerting documentation doesn't mention this, so users investigating historical alerts hit an unexplained data-disappearance wall. The underlying config is being documented by DOC-009; this item ensures the alerting section itself flags the behavior and links to the config.

## Acceptance Criteria

- [ ] Alerting section of Features.md notes that resolved alerts are automatically cleaned up after a configurable retention period (default 30 days)
- [ ] The note links (or references) the housekeeping config section covered by DOC-009
- [ ] The config key `housekeeping.ttl.resolved_alerts_days` is referenced by name
