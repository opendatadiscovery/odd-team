## Description

User spot-check on 2026-05-03 (after the `/review batch:feature/docs-editorial-audit-closeout` editorial audit ran end-to-end and missed it) surfaced **intra-page redundancy** on `docs/main-concepts.md`: the same Adapter / Plugin / Collector / Push adapter conceptual content is authored in **four distinct sections** of the same page.

### The four redundant sites

| Section | Lines | What it carries | Overlap with others |
|---|---|---|---|
| `## The architecture chain` | 11-27 | Pull-path / Push-path diagrams + full prose definitions of Data source, Adapter, Plugin, Collector, Push adapter, ODD Platform | Full overlap with sites 2, 3, 4 |
| `## Push vs Pull strategies` | 35-44 | Pull vs Push framing + "when to pick each" guidance | Re-explains Pull/Push from site 1 in different prose |
| `## Adapter, Plugin, Collector, Push adapter` | 46-57 | Same four concepts as site 1, but in table form, plus the "An adapter never runs alone…" constraint paragraph | Same definitions as site 1, table layout instead of prose |
| `## Terms & Aliases` | 99-102 | Adapter / Plugin / Collector / Push adapter rows in the alias table — **with full definitions, not alias-only** | Definitions that should have been one-line aliases pointing to the canonical home are full re-definitions |

### Reader-experience cost

A new reader hitting `docs/main-concepts.md` reads the same content **four times** and is left wondering whether they missed a distinction. The page conflates two roles:
1. **Conceptual deep dive** (what the architecture is) — site 1 + 3 (and partially 2).
2. **Alias registry** (canonical-term → alias mapping) — site 4, but it's currently doing both jobs.

The four-place pattern is exactly the failure shape `playbooks/doc-product-editorial-read.md` exists to catch — but the playbook's catalog framed `Parallel surfaces with drift` as a CROSS-PAGE shape (LSN-006 lookup-tables case), not a within-page shape. Two consecutive `/review` runs (DOC-090…093 closeout + DOC-094…101 closeout) ran the editorial audit end-to-end and missed this because the reviewer treated each H2 as a separate concern instead of asking "does this page cohere with itself?"

This finding becomes the **canonical case-law for an intra-page-redundancy failure shape** — see `playbooks/doc-product-editorial-read.md` extension proposed below.

## Failure shape

**Intra-page redundancy** (NEW failure shape; sub-class of `Parallel surfaces with drift` but specifically same-page) — extension to `playbooks/doc-product-editorial-read.md` failure-shape table needed in the same commit that ships the fix.

## Context

- **Why high (not medium)**: main-concepts.md is the canonical entry-point page for the entire ODD vocabulary — every doc-tree cross-link to "what is a Collector / Plugin / Adapter" eventually grounds out here. A reader landing on this page reads it as authoritative; four-times-redundant content tells the reader either "we don't trust you to read once" or "we don't know what we're saying" — both undermine the Principal-grade authority the page should carry. Per the world-class-doc mission, redundancy that shouldn't exist is a Cornerstone violation.
- **Why not critical**: no operator data loss / security exposure; the redundancy is a coherence problem, not a wrong-answer problem.
- **Recurring pattern**: the failure shape "intra-page redundancy" is structurally distinct from the cross-page "parallel surfaces with drift" (LSN-006) — the cross-page shape was caught and catalogued; the within-page shape was missed because the catalog had no entry for it. This is exactly the pattern LSN-011 names (the gates verify what is *authored*, not what is *redundant*) — extending the catalog is the closure.
