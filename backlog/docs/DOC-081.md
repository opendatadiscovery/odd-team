## Description

Same class of file-path-vs-URL-slug discrepancy that DOC-079 fixed for `deployment.md`. Three pages currently sit at the docs root but are nested under section pages in `SUMMARY.md`. GitBook computes URL slugs from the SUMMARY hierarchy, so the live URLs do not match the file paths:

| File on disk | SUMMARY parent | Live URL |
|---|---|---|
| `docs/lookup-tables.md` | `* [Master Data Management](master-data-management.md)` | `https://docs.opendatadiscovery.org/master-data-management/lookup-tables` |
| `docs/query-examples.md` | `* [Data Modelling](data-modelling.md)` | `https://docs.opendatadiscovery.org/data-modelling/query-examples` |
| `docs/relationships.md` | `* [Data Modelling](data-modelling.md)` | `https://docs.opendatadiscovery.org/data-modelling/relationships` |

The discrepancy is **maintainer-facing only** — operators don't see file paths, so this is not a user-visible defect. But it costs maintainer time at every grep, and (more concretely) it bit DOC-077 review on 2026-04-30: the implementer's affected-URL list cited `https://docs.opendatadiscovery.org/lookup-tables` (which 404s), not the actual GitBook URL `/master-data-management/lookup-tables`. Same DOC-079-class trap.
