## Description

DOC-081 (commit `4448b2c` on `feature/docs-summary-path-realignment`) moved
`docs/lookup-tables.md` → `docs/master-data-management/lookup-tables.md` to
align the file path with the GitBook URL slug. The move was executed cleanly
inside the `documentation` repo, but `navigation/domains/lookup-tables.md:30`
still references the old root path:

```

```

The line should read `docs/master-data-management/lookup-tables.md` so that
future navigation lookups resolve to the actual file. Other entries on that
page are still correct (the SUMMARY-placement note at line 31 already
describes the nested-under-MDM hierarchy; UI-navigation entries at line 36
reference odd-platform code paths that are unaffected).

The two sibling moves (`query-examples.md` and `relationships.md`) did **not**
introduce parallel drift — `navigation/domains/query-examples.md:34` and
`navigation/domains/relationships.md:20` both say *"Documentation: None —
feature is completely undocumented"*, which is now stale in a different way
(the docs exist) but is **pre-existing staleness independent of DOC-081**, not
a regression caused by the path realignment. Logging the doc-now-exists
update separately if/when picked up; this NAV item is scoped strictly to the
DOC-081-introduced stale path.
