## Description

Same DOC-079 / DOC-081-class file-path-vs-URL-slug drift as the three pages DOC-081 just realigned. The Use cases group nests its parent + 5 children in `SUMMARY.md`, so GitBook computes URLs under `/use-cases/{kebab-name}`, but the files all sit at the docs root with **underscored** filenames:

| File on disk | SUMMARY parent | Live URL (slug derived from SUMMARY hierarchy) |
|---|---|---|
| `docs/Use_cases.md` | top-level | `https://docs.opendatadiscovery.org/use-cases` |
| `docs/dc_data_compliance.md` | `* [Use cases](Use_cases.md)` | `/use-cases/dc-data-compliance` |
| `docs/de_deprecation.md` | same | `/use-cases/de-deprecation` |
| `docs/dq_visibility.md` | same | `/use-cases/dq-visibility` |
| `docs/viz_preparation.md` | same | `/use-cases/viz-preparation` |
| `docs/service_presales.md` | same | `/use-cases/service-presales` |

DOC-081 split this off because it carries an additional naming-convention question DOC-081 did not need to answer:

- **Filename convention**: kebab-case (`dc-data-compliance.md`) vs underscored (`dc_data_compliance.md`)? The other pages in this repo are kebab-case (`lookup-tables.md`, `query-examples.md`, `oauth2-oidc.md`, …). Use_cases is the lone underscore cluster. Renaming to kebab-case is consistent but is a content-touching change beyond the path realignment.
- **Parent file**: `Use_cases.md` has unusual capitalization + underscore. Options: keep the current name and just leave the parent at root (parent stays `Use_cases.md`, children move under `use-cases/`); rename the parent file to `use-cases.md`; or convert the parent into a `use-cases/README.md` directory landing.

This is a Cornerstone-2 / DOC-081-class hygiene fix; the answer on naming convention is what bumps it to "small" instead of "trivial".
