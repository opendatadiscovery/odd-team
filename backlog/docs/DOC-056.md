## Description

`docs/configuration-and-deployment/enable-security/authorization/permissions.md` mentions "business glossary" five times (lines 6, 32, 33, 46, 97) while describing term-related permissions — e.g., `DATASET_FIELD_ADD_TERM`, `TERM_DELETE`, `TERM_CREATE`. None of these mentions link to the canonical Business Glossary feature documentation in `Features.md` or to `main-concepts.md` where the alias is registered.

A reader landing here through search for "business glossary" gets a permissions list with no way to find out *what* the Business Glossary feature actually is. The fix is a single cross-reference near the top of the file or at the first mention.

"Business Glossary ↔ Glossary, Terms" is already an alias row in `main-concepts.md` Terms & Aliases, so no taxonomy change is needed — just a link.
