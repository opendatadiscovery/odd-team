## Description

DOC-042 shipped the `odd-collector-azure` integrations page with the full inventory (4 adapters) and **spotlight per-adapter config tables for 2 representative adapters** (`powerbi`, `blob_storage`). The remaining **2 adapters** are listed in the inventory but do not yet have inline per-field config tables.

This item completes the per-adapter user-facing reference: every type literal in `odd_collector_azure.domain.plugin.PLUGIN_FACTORY` gets a sub-section with a field-by-field table.

The 2 adapters to cover:

`azure_sql`, `azure_data_factory`.

## Implementation Notes

- `azure_sql` does NOT inherit from `AzurePlugin` (it inherits directly from `Plugin`) — its auth surface is `username` + `password` + `server` + `database` + `port`. Different from the `client_id` / `client_secret` / `domain` pattern PowerBI uses.
- `azure_sql` defaults: `encrypt="yes"`, `trust_server_certificate="no"`, `connection_timeout="30"` (note: all string-typed in the Pydantic model — that is suspicious, may warrant verification against the consuming code).
- `azure_data_factory` requires environment-variable auth via `DefaultAzureCredential` — re-state this caveat explicitly per AC, not just on the common section.

### Resolution

Two new spotlight sections added to `docs/integrations/collectors/odd-collector-azure.md` between the existing Blob Storage spotlight and the per-adapter feature matrix: `## Spotlight: Azure SQL` and `## Spotlight: Azure Data Factory`. With these two additions, all 4 PLUGIN_FACTORY adapters on the page now have inline field-by-field tables — matching the AC's "complete per-adapter reference" requirement.

`azure_sql`: 9 fields enumerated — server / port / database / username / password (required) and encrypt / trust_server_certificate / connection_timeout (optional, all string-typed). Info admonition added flagging the str-typed TLS knobs and YAML quoting consequence. Inheritance distinction (Plugin not AzurePlugin) called out in prose so operators don't expect AAD app fields.

`azure_data_factory`: 4 plugin fields + 2 nested filter fields enumerated — subscription / resource_group / factory required, pipeline_filter optional with regex include/exclude lists. Warning admonition added making the env-var-only auth posture explicit at the adapter level (not just in the page-wide Azure section), so operators don't try to wire `client_id` / `client_secret` inline and have it silently ignored.

Per-adapter feature matrix at the bottom of the page extended with two new rows: TLS knobs (`azure_sql`) and SQL-login auth (`azure_sql`). All previous matrix rows preserved.
