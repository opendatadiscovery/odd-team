## What

`S2sTokenProvider.isValidToken(String token)` dereferences the internal `s2sToken` field without a null check. If the method is called while `auth.s2s.enabled=false`, the `@Value("${auth.s2s.token:#{null}}")` binding leaves `s2sToken` null, and `s2sToken.equals(token)` on line 20 throws `NullPointerException`.

The method is `public`. Under the current `ServerHttpSecurity` wiring the code path is unreachable in production — `S2sAuthenticationFilter` is the sole caller and it is conditionally registered only when `auth.s2s.enabled=true`. But the method's public contract does not advertise that precondition, and a future caller (tests, a second filter, a refactor that moves the conditional elsewhere) can trip it easily.

## Where

`odd-platform-api/src/main/java/org/opendatadiscovery/oddplatform/auth/S2sTokenProvider.java:15-21`

```java
public boolean isValidToken(final String token) {
    if (StringUtils.isBlank(token)) {
        return false;
    }

    return s2sToken.equals(token);   // NPE when s2sToken is null
}
```

The `@Value` bindings (`s2sToken` defaults to `null`; `s2sEnabled` defaults to `false`) are declared on lines 10–13.

## Why it matters

- The NPE is unreachable today, but it is a landmine: the next refactor that calls `isValidToken` outside the conditional-filter context surfaces a crash that wouldn't appear in any test (because every test runs with `auth.s2s.enabled=true`).
- The fix is a one-line change with zero behavioral risk.
- Defensive null handling on a public method that handles authentication is a baseline expectation for a security-sensitive component.

## Suggested fix

Guard the null explicitly, or swap to a null-safe comparison:

```java
public boolean isValidToken(final String token) {
    if (StringUtils.isBlank(token) || StringUtils.isBlank(s2sToken)) {
        return false;
    }
    return s2sToken.equals(token);
}
```

Or, equivalently:

```java
public boolean isValidToken(final String token) {
    if (StringUtils.isBlank(token)) {
        return false;
    }
    return java.util.Objects.equals(s2sToken, token);
}
```

Either form preserves existing behavior for `auth.s2s.enabled=true` and returns `false` (instead of NPE) for the unreachable-today path.
