## Description

Line 139 of `odd-platform.md`, under the "Internal PostgreSQL" session provider section, shows `SESSION_PROVIDER=IN_MEMORY`. This is wrong — for the Internal PostgreSQL provider the value must be `INTERNAL_POSTGRESQL`. The sibling YAML tab (lines 101-106) correctly uses `provider: INTERNAL_POSTGRESQL`, so only the env-var tab has the bug. A user following the env-var tab ends up running the in-memory provider despite expecting PostgreSQL-backed sessions.

## Acceptance Criteria

- [ ] The env-var tab under "Internal PostgreSQL" shows `SESSION_PROVIDER=INTERNAL_POSTGRESQL`
- [ ] No other section of the page still shows `SESSION_PROVIDER=IN_MEMORY` incorrectly paired with persistent-session instructions
- [ ] The page's explanatory text (if any) does not contradict the corrected value
