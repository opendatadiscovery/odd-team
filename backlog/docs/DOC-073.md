## Description

DOC-042 shipped the `odd-collector-gcp` integrations page with the full inventory (4 adapters) and **spotlight per-adapter config tables for 2 representative adapters** (`bigquery_storage`, `gcs`). The remaining **2 adapters** are listed in the inventory but do not yet have inline per-field config tables.

This item completes the per-adapter user-facing reference: every type literal in `odd_collector_gcp.domain.plugin.PLUGIN_FACTORY` gets a sub-section with a field-by-field table.

`bigtable`, `gcs_delta`.

## Implementation Notes

- Both `bigtable` and `gcs_delta` inherit from `GcpPlugin` (which only adds `project: str`).
- `bigtable.rows_limit` defaults to `10`; the README documents this as "get combination of all types in table used across the first N rows" — surface verbatim to avoid mis-interpretation.
- `gcs_delta.delta_tables` is a list of `DeltaTableConfig` (each with `bucket`, `prefix`, optional `filter`); `gcs_delta.parameters` is the same `GCSAdapterParams` model the `gcs` plugin uses.

### Resolution

Two new spotlight sections added to `docs/integrations/collectors/odd-collector-gcp.md` between the existing GCS spotlight and the per-adapter feature matrix: `## Spotlight: BigTable` and `## Spotlight: GCS Delta tables`. With these two additions, all 4 PLUGIN_FACTORY adapters on the page now have inline field-by-field tables — matching the AC's "complete per-adapter reference" requirement.

`bigtable`: 3 fields enumerated — name / project (inherited from `GcpPlugin`) / rows_limit (optional, default 10). The README's literal phrasing "get combination of all types in table used across the first N rows" preserved verbatim because it captures the operator-actionable trade-off (sample width vs. read-unit cost) that Bigtable's schemaless layout makes load-bearing.

`gcs_delta`: 5 fields enumerated — name / project / delta_tables (required list of `DeltaTableConfig`) / parameters (optional `GCSAdapterParams`). The nested `delta_tables[*]` fields (bucket / prefix / filter.include / filter.exclude) are enumerated as separate rows in the table so an operator can build the YAML without bouncing to `plugin.py`. Info admonition called out the `schema` <-> `scheme` alias on `DeltaTableConfig` (default `gs`) for back-compat with older configs.

Per-adapter feature matrix at the bottom of the page extended with one new row: row-sampling for schema inference (`bigtable`). All previous matrix rows preserved.
