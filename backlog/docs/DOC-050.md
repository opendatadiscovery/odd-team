## Description

The platform has an in-app Integration management system for setting up data source connections, partially documented (brief mention on `main:Features.md`) but lacking setup flow detail, API reference, and the collector-config contrast. Surface:

- UI: Management → Integrations, with detail pages for each integration
- API: `IntegrationController.java` — `/api/integrations`, `/api/integrations/{integration_id}`
- Backend: `IntegrationService`

Operators using the UI to configure integrations don't know the feature exists unless they click through Management.

### Caveats surfaced (in scope)

- **Static-parameter `platform_url` default is the placeholder string `http://your.odd.platform`** — same caveat already documented in `odd-platform.md` under "Enable Alert Notifications" (DOC-060). The wizard sub-page surfaces this with a forward-pointer to that section, so an operator opening the wizard knows they must set `odd.platform-base-url` first.
- **No hot-reload of wizard manifests.** A platform restart is required to pick up new `META-INF/wizard/*.yaml` files.

### Caveats surfaced (out of scope)

- The wizard's `installed` flag is currently always `false` (set as a constant in `IntegrationMapper.java:27,30`). The flag is reserved for future state-tracking; no caveat needed today, but worth flagging if a future scanner notices the always-false value and questions it.
