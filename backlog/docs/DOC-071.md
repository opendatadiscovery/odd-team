## Description

DOC-042 shipped the `odd-collector-aws` integrations page with the full inventory (11 adapters), the `AwsPlugin` common-auth section, and **spotlight per-adapter config tables for 2 representative adapters** (`glue`, `s3`). The remaining **9 adapters** are listed in the inventory but do not yet have inline per-field config tables.

This item completes the per-adapter user-facing reference: every type literal in `odd_collector_aws.domain.plugin.PLUGIN_FACTORY` gets a sub-section with a field-by-field table.

The 9 adapters to cover:

`athena`, `dms`, `dynamodb`, `kinesis`, `quicksight`, `s3_delta`, `sagemaker`, `sagemaker_featurestore`, `sqs`.

## Context

- Discovered during DOC-042 implementation on 2026-04-26 — Phase 1 slice deliberately deferred exhaustive per-adapter detail.

## Implementation Notes

- All non-spotlighted adapters inherit `AwsPlugin` directly — the common AWS auth fields are documented once at the top of the page; per-adapter tables only need to enumerate adapter-specific fields plus call out which auth fields are typically required for that service.
- `s3_delta` is structurally similar to `s3` but uses `delta_tables: DeltaTableConfig` (single object) instead of `dataset_config` (list). Don't reuse the s3 example verbatim.
- `kinesis.aws_account_id` is **required** (not optional like the rest of the AWS auth set) — flag explicitly.
- `sagemaker.experiments` accepts an optional list of experiment names to restrict ingestion scope.
