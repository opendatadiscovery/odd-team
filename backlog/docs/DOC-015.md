## Description

Documentation still refers to the old individual collector repositories (`odd-collector`, `odd-collector-aws`, `odd-collector-gcp`) and completely omits `odd-collector-azure`. The collectors have been consolidated into the `odd-collectors` monorepo. Affected locations:

- `build-and-run-odd-collectors.md` lines 13-15 (repo list)
- `build-and-run-odd-collectors.md` line 38 (clone command pointing to old repo)
- `build-and-run-odd-collectors.md` line 82 (link to old `config_examples/` URL)
- `trylocally.md` (same class of issue — repo references)

`odd-collector-azure` contains 4 adapters (azure_sql, azure_data_factory, blob_storage, power_bi) and is never mentioned.

## Acceptance Criteria

- [ ] All references to the old individual repo URLs are replaced with `opendatadiscovery/odd-collectors` monorepo URLs
- [ ] The clone command points to the monorepo
- [ ] `config_examples/` links point to the monorepo's tree for each sub-collector (or to a top-level aggregate)
- [ ] The supported-collectors list includes all 4: generic (`odd-collector`), AWS (`odd-collector-aws`), GCP (`odd-collector-gcp`), Azure (`odd-collector-azure`)
- [ ] Both `build-and-run-odd-collectors.md` and `trylocally.md` are updated consistently
- [ ] No broken links remain after the change
