# Cornerstones — documentation pillar

The five cornerstones are the IA-level constraints the maintainer holds on every doc change. They are upstream of gates: when a structural question arises ("does this feature deserve its own page?", "where does this content belong?", "is this duplication acceptable?"), the maintainer resolves it by reading these cornerstones, not by analogy to the most recent change.

## Migration status

**This file is currently a Phase 1 scaffold.** Cornerstones 1-5 live in `CLAUDE.md` until Phase 3 of `adrs/drafts/refactor-to-pillar-architecture.md` moves them here. Until that phase ships, treat `CLAUDE.md` as the source of truth for the cornerstones; this file is the future home.

Phase 3 will move:

- Cornerstone 1 — Discoverability without context
- Cornerstone 2 — Aspect-level deep dive with single canonical source
- Cornerstone 3 — Configuration is a separate audience surface
- Cornerstone 4 — Three audiences, AI-maintained consistency
- Cornerstone 5 — One canonical home per content type
