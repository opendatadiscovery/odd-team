## Description

`application.yml` lines 165-170 define a `housekeeping` section that controls automatic deletion of stale data. None of its keys are documented:

- `housekeeping.enabled` (default: true)
- `housekeeping.ttl.resolved_alerts_days` (default: 30)
- `housekeeping.ttl.search_facets_days` (default: 30)
- `housekeeping.ttl.data_entity_delete_days` (default: 30)

Silent deletion after 30 days is a surprise for any operator who hasn't read the source — resolved alerts, search facet history, and soft-deleted entities all disappear.

## Acceptance Criteria

- [ ] A housekeeping section exists in `odd-platform.md`
- [ ] All four keys are documented with defaults, types, and env-var equivalents
- [ ] The behavior of each TTL is explained (what gets deleted at what point)
- [ ] The page notes that setting `housekeeping.enabled=false` disables all cleanup (with a caveat about storage growth)

## Review-ready notes (2026-04-24)

**What shipped** (commit `feff04d` on `feature/docs-odd-platform-config`, `documentation` repo):

1. **Section expanded**: prose lede now explains that housekeeping is a scheduled background job (15-minute cadence, ShedLock single-instance) — not just a "module" — so operators understand it runs regardless of UI activity.
2. **`housekeeping.enabled`** documented explicitly with cross-reference to the new disable-caveat admonition.
3. **Per-TTL deletion behavior** documented in prose:
   - `resolved_alerts_days` → deletes `ALERT` and `ALERT_CHUNK` rows where status ∈ `{RESOLVED, RESOLVED_AUTOMATICALLY}` and `STATUS_UPDATED_AT <= now - N days`.
   - `search_facets_days` → deletes `SEARCH_FACETS` rows past `LAST_ACCESSED_AT`.
   - `data_entity_delete_days` → deletes `DATA_ENTITY` rows with status `DELETED` plus cascading related rows (metadata, ownerships, lineage, tags, terms, alerts, messages, metrics, attachments, task runs, group relations, dataset structure, enum values).
4. **Type + default** explicit on each TTL key (integer, days, 30).
5. **Warning admonition** added for `housekeeping.enabled: false`: names the three jobs that stop, the storage-growth consequence, and the narrow conditions under which disabling is acceptable.
6. **Pinned-SHA GitHub link removed**: the prior doc linked to `odd-platform/blob/4572bba1.../application.yml#L152` — a dead reference that misdirected readers. Replaced with in-page per-key descriptions.
