## Description

`TermController.java` supports direct term-to-term linking (`addLinkedTermToTerm`, `deleteLinkedTermFromTerm`) in addition to the description-text linking that the docs describe. This is useful for glossaries with related-term relationships (e.g., "customer" ↔ "client") and is completely hidden.

Note: `TermController` also exposes Query Example associations (`createQueryExampleToTermRelationship`), but Query Examples is a standalone feature with its own work item covering it (F-038 in undocumented-features will be triaged separately — deferred from here to avoid fragmentation).

## Acceptance Criteria

- [ ] The dictionary-terms section describes direct term-to-term linking as a capability distinct from description-text mentions
- [ ] The linking API is referenced by name
- [ ] A short example shows when to use direct links vs description mentions
