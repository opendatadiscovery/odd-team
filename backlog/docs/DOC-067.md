## Description

DOC-032 reframed `Features.md` "Data Entity Report" as the "Catalog Overview page" and dropped the stale `.gitbook/img/report.png` image. The old screenshot predated the Overview reorganization (Domains, Directory, Owner association sections were added later) and showed only the per-class report card.

The new section currently has no screenshot. A fresh capture would help operators recognize the page on first visit and confirm where the per-class Entities report lives within the larger overview.

## Acceptance Criteria

- [ ] A current screenshot of the Catalog Overview page (Main search → Top tags → Domains → Entities → Directory → Owner association) is added under `docs/.gitbook/img/`
- [ ] The image is referenced from the **Catalog Overview page** section in `Features.md`
- [ ] The screenshot is taken from a deployment with at least one tag, one domain, one datasource, and a non-empty Entities report — so all six sections render with content (not empty placeholders)
- [ ] The deprecated `report.png` is removed from `.gitbook/img/` if no other page references it (grep first)

## Blocked (2026-04-30)

Examined during the 2026-04-30 batch (`feature/docs-bigbatch-2026-04-30`). The screenshot capture itself (AC1, AC2, AC3) requires a live deployment with seeded data — at minimum one tag, one domain, one datasource, and a non-empty Entities report — so the implementer can capture the page at the right moment with all six Catalog Overview sections rendering content. That capability is not available in a docs-only session; the operator with access to a populated platform UI is the right one to ship the screenshot.

Sub-AC4 ("delete deprecated `report.png` if no other page references it") is independently satisfiable and was checked: `grep -rn "report.png" docs/ --include="*.md"` returns zero hits as of commit `dd0f437`. Deleting the asset would land cleanly, but splitting the screenshot-capture work from the asset cleanup gives no operator value (the page would still be screenshot-less, just with one fewer orphaned image asset). Held the asset for now to keep the item single-actionable when an implementer with UI access picks it up.

**Unblock when**: a maintainer with access to a populated ODD Platform deployment (or sample-data Docker Compose stack with the bundled enricher) can open `/` (the Catalog Overview), capture a screenshot, and commit it under `docs/.gitbook/img/`. At that point the Features.md "Catalog Overview page" section gets the `<figure>` reference and the orphan `report.png` is deleted in the same commit.
