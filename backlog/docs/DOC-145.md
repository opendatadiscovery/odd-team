## Description

Create a new **Data Lineage** pillar under `## Features` in SUMMARY, mirroring the DOC-131 pattern (Active Platform Features pillar landing + nested sub-pages). Carve out the two existing Lineage sections from `docs/Features.md`:

- `## End-to-end Data Objects Lineage` (L94-107) — the platform's catalog-side lineage: per-entity upstream/downstream, the `lineage_depth` and `expanded_entity_ids` query parameters (currently only documented at API Reference → Lineage), and the Group lineage endpoint for DEGs and ML Experiments.
- `## End-to-end Microservices Lineage` (L108-114) — the OpenTelemetry-driven microservices lineage path through `odd-tracing-gateway` and the Push-adapter standalone shape.

Both are operator-meaningful enough to deserve dedicated pages. Today their detail is split between Features.md (high-level prose) and `developer-guides/api-reference/lineage.md` (HTTP endpoints) — neither serves as a discovery-oriented canonical home for the *feature*. The pillar landing closes that gap.

Sub-task of the umbrella **DOC-141**.
