## Description

The "Enable Data Collaboration" section describes the feature at a high level (Slack threads etc.) but omits the dedicated API endpoints:

- `/api/datacollaboration/providers/slack/channels`
- `/api/datacollaboration/providers/slack/messages`
- `/api/dataentities/{id}/messages/*`

Additionally, the Slack Events webhook URL pattern shown in the manifest template (setup instructions) may not match the actual endpoint served by the code (`/api/slack/events`). This needs verification.

## Acceptance Criteria

- [ ] The data collaboration section references the key API endpoints (at least the three listed above)
- [ ] The Slack Events webhook URL in the manifest template matches the endpoint handled by code — verified against `odd-platform-api` routes
- [ ] If a mismatch is found in the manifest template, it is corrected in the docs
- [ ] A note mentions `message-partition-period` configuration (or its current binding) if documented separately

## Implementation log (2026-04-30 — review-ready)

Batch branch: `feature/docs-bigbatch-2026-04-30` (documentation, commit `2c6add7`), paired with `feature/docs-bigbatch-2026-04-30-state` (odd-team).

### Change shipped

- Added a new `### API surface` sub-section under the existing `## Enable Data Collaboration` H2 in `docs/configuration-and-deployment/odd-platform.md`. The section enumerates the seven HTTP routes that ship with the feature, grouped by purpose:
  - **Outbound to provider — read & post** (2 routes): `GET /api/datacollaboration/providers/slack/channels`, `POST /api/datacollaboration/providers/slack/messages`.
  - **Per-entity threads & history** (4 routes): `GET /api/dataentities/{id}/messages`, `GET /api/dataentities/{id}/messages/{message_id}`, `GET /api/dataentities/{id}/channels`, `GET /api/messages/{id}/url`.
  - **Inbound webhook from Slack** (1 route): `POST /api/slack/events`.
- Confirmed the Slack-app manifest's `event_subscriptions.request_url` (`https://<ODD_PLATFORM_BASE_URL>/api/slack/events`) matches `EventApiController.java:22`'s `@PostMapping(path = "/api/slack/events")` — no manifest correction needed.
- Documented `datacollaboration.message-partition-period`'s actual binding in a footnote: it is consumed via `@Value` in `MessageTablePartitionManager.java:19`, **not** through `DataCollaborationProperties.java`. The properties class binds only `senderMessageAdvisoryLockId`, `receiveEventAdvisoryLockId`, and `sendingMessagesRetryCount`.
