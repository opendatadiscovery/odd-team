## Description

`docs/Architecture.md` is a short diagram-first page (15 lines). It bold-defines three core concepts inline — **Push-client** (line 7), **ODD Collector** (line 9), **ODDRN** (line 14) — that are now canonically defined in `main-concepts.md`. Architecture.md has no pointer to main-concepts.md, so readers arriving here for the diagram don't know there's a vocabulary page backing it.

`main-concepts.md` links *to* Architecture.md ("See Architecture.md for the diagram"). This item adds the reverse pointer so the two pages are properly linked both ways.

The one-sentence Push-client / Collector / ODDRN definitions currently on Architecture.md are compatible with main-concepts.md — they don't need to be rewritten. Only a top-of-page cross-reference is needed.

## Acceptance Criteria

- [ ] Architecture.md gains a single-line pointer at or near the top (above the diagram or immediately below it) that reads approximately: *"For the vocabulary used below (Push-client, Collector, ODDRN), see [Main Concepts](main-concepts.md)."* The exact wording can vary but must (a) name the concepts, (b) link to `main-concepts.md`, (c) use a plain markdown link (no `"mention"` shortcut)
- [ ] The three existing bold-term definitions on Architecture.md remain unchanged (this is an addition, not a rewrite)
- [ ] The diagram image reference is unchanged
- [ ] Post-merge live-site verification: WebFetch `https://docs.opendatadiscovery.org/architecture`, confirm the new pointer renders with a working internal link to `/main-concepts`

## Context

- Finding F-003 in `findings/docs-quality-duplication/2026-04-22.md`
- `main-concepts.md` is the new onboarding page created by DOC-035 (merged in PR #12, commit 762c978 on documentation main)

## Implementation Notes

- Keep the addition to a single sentence. Architecture.md is intentionally short — the diagram and three bold-term lines are the page. A long preamble would change its character.
- Match the link style used in DOC-035's main-concepts.md (plain `[Text](path.md)`, no `"mention"`), per CLAUDE.md "Documentation Authoring Rules".
