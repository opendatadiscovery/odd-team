## Description

The DOC-040 (`deployment.md` rewrite) Tracing Gateway section describes the gateway as "an optional companion component that transfers metadata to the Platform from your custom microservices" and notes that it "helps standardise the metadata against the ODD Specification". This wording is the cleaned-up version of the existing prose on origin/main (the corrupted 2026-04-* paragraph that DOC-040 had to repair) — it captures the intent but doesn't have a strong upstream Source of Truth.

The reason: the upstream `odd-tracing-gateway` repo has an empty / undiscoverable README. WebFetched 2026-04-30:
- `https://api.github.com/repos/opendatadiscovery/odd-tracing-gateway` → 200 OK, Java repo, default branch `main`, NOT archived, no description.
- `https://raw.githubusercontent.com/opendatadiscovery/odd-tracing-gateway/main/README.md` → 404.
- `https://raw.githubusercontent.com/opendatadiscovery/odd-tracing-gateway/master/README.md` → 404.

When the upstream README is populated (or a different SoT becomes available), this item ships a small revision to `deployment.md`'s Tracing Gateway section with verbatim citations from the upstream prose.
