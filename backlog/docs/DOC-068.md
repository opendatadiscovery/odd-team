## Description

`docs/trylocally.md` (top-level) is an orphan page: it is not referenced from `docs/SUMMARY.md` (which routes "Try locally" to `docs/configuration-and-deployment/trylocally.md`) and `WebFetch https://docs.opendatadiscovery.org/trylocally` returns a 404 — GitBook does not render the orphan path. The page predates the GitBook migration (single commit `853abe4 Migrate GitBook docs from ODD Platform repository`) and was never re-attached after the canonical `configuration-and-deployment/trylocally.md` was added.

Two operator-relevant problems:

1. **Stale archived-repo URLs.** The orphan still points at `github.com/opendatadiscovery/odd-collector` and `github.com/opendatadiscovery/odd-collector-aws` — both repositories were archived on 2023-11-06 (verified during the DOC-015 review on 2026-04-26). The DOC-015 batch updated only the canonical `configuration-and-deployment/trylocally.md`; the orphan retained the pre-archival URLs.
2. **Search-engine surface area.** Even though the live site 404s, the file is in `git` and crawled by GitHub's UI; an operator who Googles "ODD trylocally" can land on the orphan via `github.com/opendatadiscovery/documentation/blob/main/docs/trylocally.md` and follow the dead `odd-collector*/blob/main/README.md` links.

## Acceptance Criteria

- [ ] The orphan path is resolved by either: (a) deleting `docs/trylocally.md` outright after confirming the canonical `docs/configuration-and-deployment/trylocally.md` covers everything the orphan said; or (b) keeping the orphan only if it carries content not present in the canonical, in which case the URLs must be fixed to monorepo paths and the page added to `SUMMARY.md` so it stops being orphaned
- [ ] If deleted: `grep -rn "trylocally.md" docs/ navigation/` returns only references to `configuration-and-deployment/trylocally.md` (no dangling cross-links to the orphan)
- [ ] If modified: every outbound URL on the page passes the same archival check the DOC-015 batch did (no `odd-collector` / `-aws` / `-gcp` / `-azure` standalone-repo URLs)
- [ ] `WebFetch https://docs.opendatadiscovery.org/trylocally` continues to 404 (deletion path) or starts returning 200 with the corrected content (re-attachment path)
