## Description

Editorial audit on 2026-05-03 surfaced `docs/Data_Compliance_DS.md` as a **dead-letter parallel draft** of `docs/dc_data_compliance.md`:

### Evidence

1. **Both files carry the same H1** "# Data compliance for Data Scientists" and the same "Challenge / Solution / Scenario / Result" structure.
2. **The scenarios differ**: `dc_data_compliance.md` uses a **bookshop** scenario (`Dim_Customers, Dim_Books, Dim_Cards, Fct_transactions`); `Data_Compliance_DS.md` uses a **banking** scenario (`Dim_Customers, Fact_transactions, Dim_currency, Dim_transaction_types, Dim_rates`).
3. **`dc_data_compliance.md` is the canonical version** — referenced from SUMMARY.md line 16: `* [Data compliance for Data Scientists](dc_data_compliance.md)` and from Use_cases.md.
4. **`Data_Compliance_DS.md` is orphaned** — only mention in `grep -rn 'Data_Compliance_DS' docs/` is a **commented-out HTML block** on `Overview.md` (lines 36-37): `<!--- * [Compliance for Data Scientists](Data_Compliance_DS.md) -->`. The orphaned file was never re-linked after the migration.
5. **Capital-letter naming convention** (`Data_Compliance_DS.md`) is inconsistent with the canonical lowercase use-case file naming convention (`dc_data_compliance.md`, `de_deprecation.md`, `dq_visibility.md`, `viz_preparation.md`, `service_presales.md`) — another signal it predates the current convention.

### Most likely history

`Data_Compliance_DS.md` was the original use-case page; was renamed/replaced by `dc_data_compliance.md` (with the bookshop scenario chosen over the banking one); the original was never deleted, just commented out of any reference.

**Parallel surfaces with drift** + **dead-letter content** (per `playbooks/doc-product-editorial-read.md` failure-shape table).
