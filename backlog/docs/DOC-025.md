## Description

Features.md lines 94-97 describe tags and labels as distinct concepts ("tags for entities, labels for fields"). Labels were eliminated in v0.17.0 (Oct 2023): the labels DB table was dropped and all labels auto-converted to tags. Docs were never updated. Users following the current docs will look for a labels UI/API that doesn't exist.

## Acceptance Criteria

- [ ] "Labels" is removed from the Manual Object Tagging section
- [ ] The section clarifies that tags apply to both entities and their fields (columns)
- [ ] No other doc page still describes labels as a separate concept (spot-check via search)
- [ ] Historical note is optional — the feature merge is visible in release notes, docs don't need to carry the history
