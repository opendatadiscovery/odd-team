## Description

The `odd-collector-sdk` package enables custom collectors, and ODDRN docs mention "self-written agents" as a supported extensibility path. No developer guide exists. Would-be custom-collector authors have to reverse-engineer the SDK from existing adapters.

## Acceptance Criteria

- [ ] A new developer guide covers writing a custom collector against the SDK
- [ ] Topics: extending the base adapter class, the plugin-config pattern, implementing the metadata fetch loop, registering the adapter, packaging, deployment
- [ ] At least one minimal end-to-end example is included (skeleton adapter, minimal config, a single call to the Platform API client)
- [ ] Links to the SDK source and to representative existing adapters are provided
- [ ] DOC-014 (CollectorConfig fields) is referenced for runtime config concerns
