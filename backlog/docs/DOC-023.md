## Description

The alerts UI has three tabs — "All", "My Objects", "Dependents" — each backed by a distinct API endpoint. Docs describe alerts generically and miss the filtering structure entirely. Users with large alert volumes don't realize they can scope to their own entities or downstream dependencies.

## Acceptance Criteria

- [ ] Alerting section describes all three views with their scopes ("my" = entities the user owns; "dependents" = alerts for entities downstream of entities the user owns)
- [ ] The corresponding API endpoints (`getAllAlerts`, `getAssociatedUserAlerts`, `getDependentEntitiesAlerts`) are named
- [ ] A short guidance note helps operators pick the right view
