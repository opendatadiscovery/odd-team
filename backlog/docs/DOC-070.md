## Description

DOC-042 shipped the `odd-collector` (generic) integrations page with full structural content (inventory of all 41 adapters, common config, multi-plugin guidance, per-adapter feature matrix) plus **spotlight per-adapter config tables for 3 representative adapters** (`postgresql`, `snowflake`, `kafka`). The remaining **38 adapters** are listed in the inventory table with links to their `plugin.py` Pydantic class and `config_examples/{name}.yaml` reference, but do not yet have inline per-field config tables.

This item completes the per-adapter user-facing reference: every type literal in `PLUGIN_FACTORY` gets a sub-section with a field-by-field table (Name, Type, Required, Default, Description) extracted from the Pydantic model in `plugin.py` and cross-checked against `config_examples/{name}.yaml`.

The 38 adapters to cover (alphabetical):

`airbyte`, `cassandra`, `ckan`, `clickhouse`, `cockroachdb`, `couchbase`, `cubejs`, `databricks`, `dbt`, `druid`, `duckdb`, `elasticsearch`, `feast`, `fivetran`, `hive`, `kubeflow`, `metabase`, `mlflow`, `mode`, `mongodb`, `mssql`, `mysql`, `neo4j`, `odbc`, `odd_adapter`, `opensearch`, `oracle`, `presto`, `redash`, `redshift`, `scylladb`, `singlestore`, `sqlite`, `superset`, `tableau`, `tarantool`, `trino`, `vertica`.

## Implementation Notes

- Group by adapter category (databases, BI tools, streams, MLOps, search, file/document stores) for scannability — the alphabetical list above is the gap inventory, not the recommended page order.
- Some plugin classes inherit from `DatabasePlugin` which itself inherits from `WithHost` + `WithPort` — surface the inherited fields explicitly in the table, don't expect a reader to follow the inheritance chain in `plugin.py`.
- Where `config_examples/{name}.yaml` and `plugin.py` disagree (drift), trust `plugin.py` and log the example drift as a follow-up against `odd-collectors`.
- This item is the natural pair to DOC-016 (which is now developer-facing — how to author a new adapter, SDK protocol). Don't duplicate DOC-016's territory.
