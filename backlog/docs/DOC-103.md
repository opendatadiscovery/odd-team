## Description

Editorial audit on 2026-05-03 surfaced `docs/Overview.md` as a **dead-letter parallel surface** to `docs/README.md`:

### Evidence

1. **Both pages have the same H1** ("# Overview") and similar marketing landing-page content about ODD's vision, audience, and feature pointers.
2. **`README.md` is the canonical landing**: `docs/SUMMARY.md` line 3 has `* [Overview](README.md)` — README.md is what an actual user lands on at the top of the table of contents. `Overview.md` does NOT appear anywhere in SUMMARY.md.
3. **`Overview.md` is referenced from no live page** — `grep -rn 'Overview.md' docs/` returns only SUMMARY.md (which actually points at `README.md`, not `Overview.md`). Overview.md is unreachable from the published site's navigation.
4. **The two files have diverged**:
   - README.md uses `.png` images (`audience.png`, `alltogether.png`) that exist in `.gitbook/img/`.
   - Overview.md uses `.svg` images (`audience.svg`, `alltogether.svg`, `onboarding.svg`) — **none of these .svg files exist** in `.gitbook/img/` (verified via `ls .gitbook/img/` — only the .png variants are present).
   - README.md lists 5 bullet points about ODD; Overview.md lists 6 with checkmark emojis (✔️) and adds "AI-based UX".
   - Overview.md line 7 has stray slash: `/The open-source Data Discovery Platform...` (typo).
   - Overview.md has a different structure with "Features and use cases" combined section.
5. **Both files have dead HTML comment blocks** at the bottom (Overview.md lines 32-38 includes a commented-out reference to `Data_Compliance_DS.md` — see DOC-104).

### Most likely history

`Overview.md` was likely an earlier draft (with planned `.svg` assets that never shipped) that GitBook editor migration replaced with `README.md` (using existing `.png` assets). The old file was never deleted.

## Failure shape

**Parallel surfaces with drift** + **dead-letter content** (per `playbooks/doc-product-editorial-read.md` failure-shape table).

## Acceptance Criteria

- [ ] **Decide**: delete `docs/Overview.md` (recommended — the file is orphaned, has broken images, and is drifted from README.md), OR formally consolidate by deciding which content belongs on README.md and migrating + deleting the old file.
- [ ] If deleting: `git rm docs/Overview.md` and verify with `grep -rn 'Overview.md' docs/` that no inbound link breaks. SUMMARY.md is not affected (already points at README.md).
- [ ] Optional cleanup: same commit also removes the dead HTML comment blocks at the end of `docs/README.md` (lines 39-42) — they reference an abandoned content idea ("Open Data Discovery makes all your data entities reliable, observable…") and clutter the source.
- [ ] Live-site verification (Gate 8 / `/review`): WebFetch `https://docs.opendatadiscovery.org/` and verify the landing page (rendering README.md) is unchanged.
- [ ] Sources footer: `Sources: none (dead-letter cleanup, no factual claim)`.

## Context

- **Why medium (not low)**: a dead-letter parallel surface is a maintenance trap — a future maintainer might edit `Overview.md` thinking they're updating the landing page; their changes never ship to readers. Also broken `.svg` references would render as broken images if anyone ever did link to it.
- **Why not high**: the file is currently unreachable, so no live reader sees the broken images today.
- **Recurring pattern**: every time a doc gets restructured via GitBook editor, the old file may survive in source. Worth a Gate-7 extension: SUMMARY.md sync check should also flag any `.md` file in `docs/` that is NOT referenced from SUMMARY.md.
