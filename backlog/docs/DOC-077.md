## Description

Two doc surfaces (`docs/lookup-tables.md` lines 118-126 and `docs/configuration-and-deployment/odd-platform.md` lines 57-59) describe the `spring.custom-datasource.*` keys with a "default value is X" framing — for example:

> `spring.custom-datasource.url`: ... Default value is `jdbc:postgresql://127.0.0.1:5432/odd-platform`.

Verified via `R2DBCConfiguration.java:56-58` that the `@Value` declarations are:

```java
@Value("${spring.custom-datasource.url:}") final String url,
@Value("${spring.custom-datasource.username:}") final String username,
@Value("${spring.custom-datasource.password:}") final String password,
```

The `:` after each key with no value means the literal `@Value` default for each `spring.custom-datasource.*` key is **the empty string**, not the JDBC URL / username / password values the doc lists. The values shown (`jdbc:postgresql://127.0.0.1:5432/odd-platform`, `odd-platform`, `odd-platform-password`) are actually the defaults of the **primary `spring.datasource.*` keys** declared in `application.yml`, which `R2DBCConfiguration` falls back to when the `spring.custom-datasource.*` keys are unset.

The values are correct for what an operator would observe with a default deployment; the framing — calling them "default values" of the `spring.custom-datasource.*` keys — is misleading. An operator reading the doc gets the right behaviour but the wrong mental model of where the values come from; a sysadmin debugging a connection issue (e.g., overrode `spring.datasource.url` and now wonders why `spring.custom-datasource.url` is also using the new value) would be misled into looking for a `spring.custom-datasource.*` cascade that doesn't exist.

## Acceptance Criteria

- [ ] `docs/lookup-tables.md` lines 118-126 ("Direct database access" code block + the YAML comments) — reframe the inline comments to make the fallback explicit. Suggested phrasing: `# unset by default; falls back to spring.datasource.url, which itself defaults to jdbc:postgresql://127.0.0.1:5432/odd-platform`.
- [ ] `docs/configuration-and-deployment/odd-platform.md` lines 57-59 — same reframing. The three bullets currently say "Default value is X"; change to "Falls back to `spring.datasource.{url,username,password}` when unset; the platform's primary `spring.datasource.*` defaults are X" (or equivalent that names the cascade).
- [ ] Sweep confirms no other "Default value is" / "the default value is" sentences attached to `spring.custom-datasource.*` keys remain in the doc tree.
