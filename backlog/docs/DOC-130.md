## Description

Editorial audit at post-merge state of `feature/docs-coherence-followups-2026-05-05` ran an end-to-end read of `docs/Architecture.md` (the page that DOC-099 Phase A touched at L11). The L13 ODDRN paragraph carries a verb-form typo missed by every prior batch:

```
**ODDRN** (Open Data Discovery Resource Name) is a unique resource name that identifies entities such as data sources, data entities, dataset fields etc. It is used to built lineage and update metadata.
```

`built` is the past participle / past tense; the correct form here is the bare infinitive `build` (the construction is `is used to {INFINITIVE}`). A reader scanning the page sees a grammar slip on the canonical Architecture page — undermines the Principal-grade authority the page should carry.

### Why this slipped past prior batches

The L13 ODDRN paragraph was not modified by DOC-099 Phase A (which touched L11 — Collector category fix) or by any prior batch. DOC-113's pattern-driven sweep targeted specific known typos (`priviligies`, `crushes`, `invironment`, `Platfrom`, `recomendations`); `to built` is not in that pattern set. The editorial audit's end-to-end read catches this class of typo — exactly the recurring pattern A12 case-law names (`pillars/documentation/quality-framework.md` A12 audit cue: "Read the page end-to-end as a published manual. Note every typo, broken markdown, missing space, escaped backslash that should render literal").
