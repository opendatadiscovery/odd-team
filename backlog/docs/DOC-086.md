## Description

DOC-083 Phase D ("spot-check `docs/genai.md` for config-reference fragments that should be cross-linked to `configuration-and-deployment/odd-platform.md`") surfaced the following Cornerstone 5 / Gate 10 drift:

`docs/genai.md` carries a `## Configuration` H2 (~lines 26-59) that documents the same three `genai.*` config keys (`enabled`, `url`, `request_timeout`) the canonical home `docs/configuration-and-deployment/odd-platform.md` `## GenAI Configuration` H2 already documents. Both surfaces ship the same silent-misconfig warning admonition. The two were authored together under DOC-045, but the result is parallel content of the same content type ("configuration reference") on a feature page when a canonical home exists.

The genai.md version has one piece of content the canonical home may not currently carry: the **field-default vs application.yml-default educational table** (Java `String`/`int` field defaults vs commented-out application.yml lines that look like defaults). That asymmetry is the operator-actionable explanation of *why* the keys must be set explicitly. Before collapsing genai.md, verify the canonical home carries this educational content and migrate if missing.

This is the same drift class as DOC-083 (content of a recognizable type embedded on a feature page when the canonical home exists), applied to **configuration reference** rather than API reference. Cornerstone 5's content-type table at `pillars/documentation/canonical-homes.md` names `configuration-and-deployment/odd-platform.md` as the canonical home for configuration reference; feature pages link, do not embed.

## Context

- **Discovered during DOC-083 Phase D spot-check (2026-05-03).** DOC-083 explicitly delegated this class of finding ("Configuration reference fragments — log a separate DOC-NNN per content type as needed") rather than expanding scope.
- **Why DOC-045 shipped both surfaces**: the genai.md version was authored as part of the GenAI feature page (audience: developers exploring the feature); the odd-platform.md version was added as the operator-facing config home (audience: operators tuning the deployment). At the time, Cornerstone 5 / Gate 10 had not yet been codified (the rule landed 2026-04-30 in the same conversation that surfaced DOC-083). DOC-045 is reviewed-approved and `done`; this item is the post-codification homing fix.
- **Scope is genuinely small**: a single feature page, a single H2, a single canonical home. The educational-caveat migration (Phase A) is the only judgment call — if it's already on the canonical home, Phase B is a 5-minute edit.
- **Why medium priority and not low**: the parallel surface produces real audience harm — an operator reading the canonical home gets the YAML/env tabs but may miss the field-default-vs-yml-default trap; a developer reading the feature page gets the trap but may miss the operator's deployment-flow context. Single canonical home with the richest content fixes both.
