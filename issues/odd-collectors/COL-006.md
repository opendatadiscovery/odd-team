## What

The reference YAML for the `cockroachdb` adapter in the generic `odd-collector` package is named `cocroachdb.yaml` — note the missing `k` between `roac` and `db`. The type literal in `PLUGIN_FACTORY` and the file's contents are correct (`type: cockroachdb`); only the filename is misspelled.

Operators looking for `cockroachdb.yaml` (the natural search) won't find it; they need to know the typo to discover the file.

## Where

- `odd-collector/config_examples/cocroachdb.yaml` — current filename.
- `odd-collector/odd_collector/domain/plugin.py:297-300` — `CockroachDBPlugin` registered with the correct `type: Literal["cockroachdb"]`.
- `odd-collector/odd_collector/domain/plugin.py:383` — `PLUGIN_FACTORY` key is the correct `"cockroachdb"`.

## Why it matters

- Discoverability gap — `ls config_examples/ | grep cockroach` returns nothing on this repo today; operators expect the filename to match the type literal.
- Severity is **low** — the file works as-is and operators who navigate the directory listing will eventually find it; this is a pure naming hygiene fix.

## Suggested fix

Rename the file:

```bash
git mv odd-collector/config_examples/cocroachdb.yaml \
        odd-collector/config_examples/cockroachdb.yaml
```

No content changes needed.
