## Description

`application.yml` lines 165-170 define a `housekeeping` section that controls automatic deletion of stale data. None of its keys are documented:

- `housekeeping.enabled` (default: true)
- `housekeeping.ttl.resolved_alerts_days` (default: 30)
- `housekeeping.ttl.search_facets_days` (default: 30)
- `housekeeping.ttl.data_entity_delete_days` (default: 30)

Silent deletion after 30 days is a surprise for any operator who hasn't read the source — resolved alerts, search facet history, and soft-deleted entities all disappear.

## Acceptance Criteria

- [ ] A housekeeping section exists in `odd-platform.md`
- [ ] All four keys are documented with defaults, types, and env-var equivalents
- [ ] The behavior of each TTL is explained (what gets deleted at what point)
- [ ] The page notes that setting `housekeeping.enabled=false` disables all cleanup (with a caveat about storage growth)
