## Description

Editorial audit on 2026-05-03 surfaced a recurring **broken markdown** pattern on `docs/developer-guides/github-organization-overview.md`. Every repository entry uses this shape:

```
**odd-platform** \[l[ink](https://github.com/opendatadiscovery/odd-platform)]
```

The intent is clearly to render `**odd-platform** [link]` where `[link]` is a clickable hyperlink. But the actual markdown renders ambiguously:
- `\[l` is escaped `[l`, then `[ink](url)` is a markdown link with text "ink", then `]` is a literal `]`.
- Result on the live site: `**odd-platform** [link]` with `link` rendered with `[` before `l` and `]` after, but with the `[ink]` part being the actual hyperlink target. Visually confusing.

Some entries on the same page use a **different** broken pattern that's slightly closer to working — for example line 32:
```
**odd-collector-azure** [\[link\]](https://github.com/opendatadiscovery/odd-collectors/tree/main/odd-collector-azure)
```

This second pattern is the **intended** one — `[\[link\]](url)` renders as a hyperlink with display text `[link]`. But the page mixes both patterns inconsistently across entries.

### Affected entries

The broken pattern `\[l[ink](url)]` appears on lines 7, 11, 15, 19, 23, 26, 29, 35, 39, 43, 47, 51, 55, 59, 63 (15 entries).
The correct pattern `[\[link\]](url)` appears only on line 32 (1 entry).

## Failure shape

**Outdated baked-in assumption** + **rendering defect** (per `playbooks/doc-product-editorial-read.md` failure-shape table).

## Acceptance Criteria

- [ ] Replace every `\[l[ink](url)]` on `docs/developer-guides/github-organization-overview.md` with the canonical `[\[link\]](url)` form (matching the correct line 32 pattern).
- [ ] Decide on a single, consistent style for "view repo" links across the page. Options:
  - `[\[link\]](url)` — the existing intended form (preserve).
  - `[`Repo`](url)` — simpler with no brackets in display text.
  - Make the **bold name** itself the link: `[**odd-platform**](url)` — most idiomatic markdown.
  - Drop the link decoration entirely and put the URL in a sentence: "**odd-platform** ([github.com/opendatadiscovery/odd-platform](url))".
- [ ] Live-site verification (Gate 8 / `/review`): WebFetch `https://docs.opendatadiscovery.org/developer-guides/github-organization-overview` and confirm every entry renders with the canonical link decoration.
- [ ] Sources footer: `Sources: none (markdown rendering fix, no factual claim)`.

## Context

- **Why low**: the links are functional today (the underlying `[ink](url)` markdown link form does click through to the right URL). The defect is cosmetic.
- **Why log at all**: per the world-class doc mission, a doc page that mixes two markdown patterns inconsistently is a maintenance liability — future maintainers will copy whichever pattern they see first, perpetuating the inconsistency. And the rendered result looks unprofessional.
