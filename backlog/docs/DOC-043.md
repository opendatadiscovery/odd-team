## Description

Features.md lines 37-43 describe lineage support for 4 types (Datasets, Data providers, ETL/ML jobs, ML models and BI dashboards). Code supports all 9 entity classes including `DATA_CONSUMER`, `DATA_INPUT`, `DATA_ENTITY_GROUP`, `DATA_RELATIONSHIP`. The upstream/downstream APIs also support `lineage_depth` and `expand_ids` parameters not mentioned in docs. Group lineage has its own API.

## Acceptance Criteria

- [ ] The lineage section describes the full entity-class scope (or a general "supports all entity classes" statement with a pointer to the entity-class reference)
- [ ] `lineage_depth` and `expand_ids` query parameters are documented
- [ ] The group-lineage API endpoint is mentioned
- [ ] The description doesn't re-enumerate types in a way that will rot again — prefer "all entity classes" with a link over a hand-maintained list
