## Description

Editorial audit at post-merge state of `feature/docs-doc131-pillar-relpath-fix-2026-05-06` ran an end-to-end read of `docs/active-platform-features/genai.md` (the migration target of DOC-131 Phase B.5 — `git mv docs/genai.md docs/active-platform-features/genai.md`). The page's `## Known limitations` section (L99-106) carries a bullet whose intra-page reference is dangling:

```
* **Three `@ConfigurationProperties` defaults are unfortunate when only `genai.enabled` is set.** `url` defaults to empty string, `request_timeout` defaults to `0` (= immediate timeout). The platform will accept the misconfiguration at startup and fail at first request. **The warning admonition above is the operator-facing fix**; the upstream-side improvement (defaults declared in `GenAIProperties` or a startup validation) is queued as a separate platform issue.
```

The bullet says "The warning admonition above is the operator-facing fix" — but the page has **no warning admonition above this bullet**. The only admonition above is an `info` hint at L65-67 about external-service authentication ("The platform sends **no authentication** to the external service…"), which is unrelated to the unset-defaults caveat the bullet is trying to mitigate.

A reader following the bullet's pointer ("the warning admonition above") scrolls upward looking for the operator-facing fix and finds only the unrelated auth hint, leaving the unset-defaults caveat with no actionable mitigation on this page (the canonical operator-side fix lives on the configuration reference at `configuration-and-deployment/odd-platform.md#genai-configuration` — the bullet should be pointing there or the warning admonition should be restored on this page).

### Why this slipped past prior batches

The dangling reference was introduced by **DOC-086** (commit `a32c1e9`, closed 2026-05-04 post-merge), which collapsed `genai.md ## Configuration` from a parallel surface (containing the warning admonition + commented-out-line caveat) to a one-paragraph pointer at the canonical-home `configuration-and-deployment/odd-platform.md#genai-configuration`. DOC-086's collapse correctly removed the duplicated configuration content but did **not** rewrite the `## Known limitations` bullet that referenced the now-removed admonition. The L101 bullet went from "the warning admonition above is the operator-facing fix" → still pointing to a warning admonition that no longer existed on the page.

**DOC-131 Phase B.5** (commit `e95ed44`) moved the page from `docs/genai.md` to `docs/active-platform-features/genai.md` via `git mv` — preserved history, but also preserved the dangling reference verbatim. The migration verification (per the prior `/review` Phase B.5 PASS criterion: "history preserved per the diff stat showing `docs/{ => active-platform-features}/genai.md`") did not include intra-page reference re-validation; the editorial audit's per-page walk catches this on the new path's first audit pass.

Pre-DOC-086 state (verified via `git show a32c1e9~1:docs/genai.md`):
```
{% hint style="warning" %}
**Java field defaults bite when `genai.enabled=true` is set without the other two.** `genai.url` is a `String` field with no annotation default — its Java field default is the empty string. `genai.request_timeout` is a primitive `int` — its Java field default is `0`, and `WebClientConfiguration` builds the WebClient with `Duration.ofMinutes(0)` which is **immediate timeout**: any request will fail before the external service has a chance to respond.

The values shown in `application.yml` lines 19-20 (`http://localhost:5000` and `2`) are **commented out** — they look like defaults but are not. Always set both `genai.url` and `genai.request_timeout` explicitly when enabling.
{% endhint %}
```

Post-DOC-086 + DOC-131 state (verified via Read of `docs/active-platform-features/genai.md` L65-106): only the L65 `info` admonition (about no-auth) remains on the page; the L101 bullet's "warning admonition above" has no referent.
