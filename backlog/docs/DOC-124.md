## Description

Editorial audit during `/review batch:feature/docs-fidelity-batch` (2026-05-04 post-merge) — page-level read of the **use-case subtree** (`dc_data_compliance.md`, `de_deprecation.md`, `dq_visibility.md`, `service_presales.md`, `viz_preparation.md`, all listed `Use_cases.md` line targets) — surfaced an **A12 (Prose and rendering) residue** that DOC-113's cross-page sweep partially handled but did not eliminate. DOC-113 covered viz_preparation.md (lines 15 + 22-23 indentation), de_deprecation.md (line 16 double-space), dq_visibility.md (line 9), and service_presales.md (lines 3, 23). The pre-existing defects below were not in DOC-113's catalogue and remain on disk after the batch merged.

### Why this slipped past DOC-113

DOC-113's defect-class catalogue (typo set + grammar set + formatting set + reader-flow D-class) was scoped to specific lines pre-identified during the 2026-05-03 audit. The use-case subtree had **viz_preparation.md** + **de_deprecation.md** + **dq_visibility.md** + **service_presales.md** in the affected_files but the line-level grep that produced DOC-113's catalogue used `priviligies|crushes|invironment|Platfrom|recomendations|PosgreSQL` — the queries did not include double-space (`  `) or `recourses`-class typos. DOC-113's own implementation log on the Caveat about U+00A0 NBSP-prefixed lines confirms the sweep was running pattern-by-pattern, not full-page-as-a-reader.

This is exactly the failure shape `retrospectives/LSN-011-doc-product-coherence-not-self-detecting.md` warns about: pattern-based sweeps catch what their patterns name; the editorial audit fills the gap by reading the page as a reader would. DOC-124 is the residue this audit caught.

### Failure shape

**A12 (Prose and rendering) — prose polish + outdated baked-in formatting** (per `pillars/documentation/quality-framework.md`).

DOC-113's pattern-driven run was correct for the catalogued defects; what it missed is the per-page final read. Worth a Gate-12 / playbook extension in `playbooks/doc-product-editorial-read.md`: after every cross-page sweep ships, run a per-affected-page editorial read as a parallel-track audit.
