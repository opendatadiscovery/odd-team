## Description

`permissions.md` lists an outdated permission set. The current `Permission` enum in `components.yaml` diverges in two directions:

**Missing from docs (~20):**
- `DATA_ENTITY_ALERT_CONFIG_UPDATE`, `DATA_ENTITY_ATTACHMENT_MANAGE`, `DATA_ENTITY_STATUS_UPDATE`
- `DATASET_FIELD_DESCRIPTION_UPDATE`, `DATASET_FIELD_INTERNAL_NAME_UPDATE`, `DATASET_FIELD_TAGS_UPDATE`, `DATASET_FIELD_ADD_TERM`, `DATASET_FIELD_DELETE_TERM`
- All `QUERY_EXAMPLE_*` (7 permissions)
- All `LOOKUP_TABLE_*` (9 permissions)
- `OWNER_RELATION_MANAGE`

**Ghost entries (in docs but not in code):**
- `LABEL_CREATE`, `LABEL_UPDATE`, `LABEL_DELETE` — removed with labels→tags merge in v0.17.0
- `DATASET_FIELD_INFO_UPDATE` — split into `_DESCRIPTION_UPDATE` + `_INTERNAL_NAME_UPDATE`
- `DATA_ENTITY_GROUP_DELETE` — not in the Permission enum
