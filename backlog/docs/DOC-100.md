## Description

Editorial audit on 2026-05-03 surfaced a formatting defect in the **Logging Settings Configuration** section's env-var tab on `docs/configuration-and-deployment/odd-platform.md`:

Line 1004 (inside the `Environment Variables` tab of the logging code block):

```
LOGGING_LEVEL_ORG_OPENDATADISCOVERY_ODDPLATFORM_DATACOLLABORATION: info    LOGGING_LEVEL_ORG_OPENDATADISCOVERY_ODDPLATFORM_SERVICE_INGESTION: info
```

Two distinct env-var assignments are concatenated on a single line (separated by spaces, not newlines). An operator copy-pasting the example would set `LOGGING_LEVEL_ORG_OPENDATADISCOVERY_ODDPLATFORM_DATACOLLABORATION` to the literal string `info    LOGGING_LEVEL_ORG_OPENDATADISCOVERY_ODDPLATFORM_SERVICE_INGESTION: info` and never set the second key — silent misconfiguration.

Minor: the whole "Environment Variables" tab uses the YAML-key colon syntax (`KEY: value`) rather than the bash export syntax (`KEY=value` or `export KEY=value`) that other env-var tabs on the same page use (e.g., `SESSION_PROVIDER=IN_MEMORY` at line 135, `NOTIFICATIONS_ENABLED=true` at line 434). For consistency, the logging tab should use the same `KEY=value` form.

## Why this slipped past prior batches

The Logging Settings section was likely authored once and not surveyed for cross-section consistency. No scanner today checks env-var-tab formatting consistency across the page.

Editorial-audit case-law for `playbooks/doc-product-editorial-read.md` — failure shape: Outdated baked-in assumption / Half-finished narrative (two lines collapsed during an edit and never re-separated).
