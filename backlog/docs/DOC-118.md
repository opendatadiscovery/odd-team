## Description

Editorial audit during `/review batch:feature/docs-editorial-audit-closeout` (2026-05-04) surfaced `docs/developer-guides/how-to-contribute.md` as a **half-finished narrative**.

The page's introduction (line 9) says:

> In this article you will find instructions on how to create an issue, open a PR or reach us via our Slack.

The body delivers two of the three paths:

1. **Creating GitHub issues** — `### Creating GitHub issues` H3 + screenshot + walkthrough. ✓
2. **Forking a repository** — `### Forking a repository` H3 + clone command + remote/branch setup. Ends with "That's it! You are good to go to make changes!" ✓

Missing:

3. **Opening a PR** — no instructions on `git push origin issue/<issue_num>`, opening the PR on GitHub, the PR title/description conventions, the review/merge process, or what reviewers will look for.
4. **Reaching the team via Slack** — no Slack workspace/invite link, no channel name, no description of what kinds of questions go there vs. on a GitHub issue.

A first-time contributor reading this page expecting the three promised paths gets stranded after forking — the page's exit point is "make changes" with no continuation. They have to guess how to open a PR (most contributors will figure it out from GitHub's own UI, but the doc abdicates) and have no path to community contact at all.

## Why this slipped past prior batches

The page has been on the doc tree since the early ODD docs era. Per `git log`, prior touches have been minor edits (typo fixes, image swaps) — no editorial audit ever reconciled the body against the introduction's three-promise commitment. Cross-page A7 (reader-flow integrity) was not part of the audit cue catalog before the framework codification (`pillars/documentation/quality-framework.md`, 2026-05-03) that introduced "half-finished narrative" as a derived failure shape under axis A7.

DOC-113 (Cross-page prose / grammar batch) covers minor grammar fixes on this page (lines 7, 38, 49) but does **not** cover the structural completeness gap — DOC-113's scope is A12 (prose / rendering), this finding is A7 (reader-flow integrity).
