## Description

The email notification configuration in `odd-platform.md` uses a key that does not exist in the code, and omits four keys that do exist.

Documentation claims (lines 244, 270, 293 of `odd-platform.md`):
- `notifications.receivers.email.smtp` — **does not exist**
- Lists 5 keys: `sender`, `password`, `smtp`, `port`, `notification.emails`

Actual config (`application.yml` line 188):
- Correct key is `notifications.receivers.email.host`, not `smtp`
- Full list of 8 keys: `sender`, `password`, `host`, `port`, `protocol`, `smtp.auth`, `smtp.starttls`, `notification.emails`

The env var `NOTIFICATIONS_RECEIVERS_EMAIL_SMTP` currently shown in the docs binds to a nonexistent property. Users following the docs cannot configure email notifications.

## Acceptance Criteria

- [ ] All occurrences of `notifications.receivers.email.smtp` in `odd-platform.md` are replaced with `notifications.receivers.email.host`
- [ ] The env-var table shows `NOTIFICATIONS_RECEIVERS_EMAIL_HOST` (not `...SMTP`)
- [ ] The YAML example block structures the config correctly with `host:` at the same level as `port:`
- [ ] Four previously undocumented keys are added with descriptions and env-var equivalents: `protocol`, `smtp.auth`, `smtp.starttls`, and the correct rendering of `host`
- [ ] A working end-to-end YAML example is included (Gmail or a generic SMTP server)

**In-scope amendments required** (F-CAV-001 in `findings/docs-accuracy-integration-caveats/2026-04-23-reaudit-critical-odd-platform-config.md`):

- `NotificationConfiguration.java:51-72` — `JavaMailSenderImpl` leaves the following unset; all are operator-visible silent failure modes under Gate 5 (Unset-parameter audit):
  - `mail.smtp.connectiontimeout`, `mail.smtp.timeout`, `mail.smtp.writetimeout` — SDK default is **infinite**; an unreachable SMTP server hangs the notification thread indefinitely
  - `mail.smtp.ssl.enable` — SDK default is `false`; implicit-TLS SMTP servers (e.g. port 465, many corporate relays) are unreachable — only STARTTLS is supported
  - `mail.smtp.ssl.trust` — SDK default is JVM truststore only; internal / self-signed CAs fail unless operator extends the JVM truststore — **no ODD config key exposes this**
  - `mail.smtp.ssl.checkserveridentity` — hostname verification behavior not operator-controllable
- `EmailNotificationSender.java:47` — `MimeMessageHelper(mimeMessage)` constructor takes no charset; non-ASCII subjects/bodies may be mangled depending on default charset
- `EmailNotificationSender.java:44-61` — `send()` loops over `notificationsEmails` and calls `emailSender.send()` per recipient; on recipient N failure, the exception wraps as `RuntimeException` (line 59) and **remaining recipients after N do not receive the message** — silent partial delivery

**Classification**: in-scope for DOC-005 (DOC-005's remit was "document email notifications config completely"; defaults and failure modes are part of complete coverage under Quality Bar #3 + #5).

**Action**: Amend `feature/critical-odd-platform-config` (documentation) with a "Known limitations" subsection in the email notifications area. Must be signed off via `/review` in a separate session before this item can flip to `done`.
