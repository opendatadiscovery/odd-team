## Description

Two related inaccuracies in the "Data Entity Report" section of Features.md:

1. The page description (lines 108-116) treats the "report" as the entire main page. Today, `Overview.tsx` renders 6 sections — MainSearch, TopTagsList, Domains, DataEntitiesUsageInfo, Directory, OwnerAssociation. The "report" is now just one section (`DataEntitiesUsageInfo`) inside a richer overview. The screenshot `report.png` is also stale.

2. Lines 112-113 list counters for "Datasets, Quality Tests, Data Inputs, Transformers and Groups" (5). Code has 9 entity classes: DATA_SET, DATA_TRANSFORMER, DATA_TRANSFORMER_RUN, DATA_QUALITY_TEST, DATA_QUALITY_TEST_RUN, DATA_CONSUMER, DATA_INPUT, DATA_ENTITY_GROUP, DATA_RELATIONSHIP. DATA_CONSUMER, DATA_RELATIONSHIP, and the "run" types are missing.

## Acceptance Criteria

- [ ] The section is retitled or restructured so "Overview page" and "entity report" are distinguished — the report is a component inside the overview, not the whole page
- [ ] The 6 overview sections are either named or covered briefly
- [ ] The entity-class counter list either matches all 9 classes or states that the overview shows all registered classes (to avoid the list going stale again)
- [ ] The screenshot (`report.png`) is flagged for replacement — either updated or replaced with a current capture
- [ ] Separate consideration: if the screenshot replacement is out of scope for this work item, log it as a follow-up item
