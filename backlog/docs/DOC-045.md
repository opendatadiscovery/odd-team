## Description

The platform ships with a GenAI integration that proxies natural-language questions to an external AI endpoint. Surface:

- `GenAIController.java` — `POST /api/genai/ask`
- `GenAIService` + `GenAIProperties`
- Config (`application.yml` lines 17-20): `genai.enabled` (default: false), `genai.url`, `genai.request_timeout`

Zero documentation exists. Users don't know the feature is shippable, how to enable it, or what API contract the external AI endpoint must implement.

## Acceptance Criteria

- [ ] A new `genai.md` documentation page covers what the feature does (proxy natural-language questions to an external AI service)
- [ ] Configuration section documents `genai.enabled`, `genai.url`, `genai.request_timeout` with defaults and env-var equivalents
- [ ] The external AI endpoint API contract is specified (request/response shape expected by `GenAIService`)
- [ ] The `/api/genai/ask` endpoint is referenced
- [ ] UI entry point is described (how users access the "ask" affordance in the platform UI, if any)
- [ ] Features.md references the page
- [ ] SUMMARY.md includes the new page
- [ ] Platform config page links to the GenAI page from the relevant config section

- **Caveats surfaced (in-scope, shipped on the new pages):**
  - **`genai.url` defaults to empty string** at the Java field level (no `@ConfigurationProperties` default). When `enabled=true` but `url` unset, the WebClient builds with empty baseUrl. Documented in genai.md Configuration warning admonition + Known limitations + odd-platform.md GenAI Configuration warning admonition.
  - **`genai.request_timeout` defaults to 0** at the Java primitive level. `Duration.ofMinutes(0)` means immediate timeout — every request fails before the external service can respond. Same admonitions cover this.
  - **`application.yml` commented examples** (`http://localhost:5000`, `2`) look like defaults but are not — same misframing class as DOC-077 and DOC-008.
  - **No authentication to external service** — the platform sends no headers. Operator must enforce auth at the network layer.
  - **No retry loop** — single attempt per /api/genai/ask call.
  - **request_timeout is in minutes** (not seconds, not ms). Minimum effective value is 1.
  - **WebClient is built once at startup** — changing url or request_timeout requires Platform restart.
  - **API-only feature, no UI affordance today** — generated TypeScript clients exist but no view/button calls them.
- **Caveats out-of-scope (logged as upstream issue draft):**
  - `issues/odd-platform/PLT-008.md` (medium severity) — `GenAIProperties` lacks defaults for `url` and `requestTimeout`; same silent-misconfiguration class as DOC-008's MinIO region miss and DOC-077's spring.custom-datasource framing. Two suggested fixes in the draft (Option A: declare field defaults; Option B: fail-fast validation via JSR-303).
