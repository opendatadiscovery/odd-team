## Description

Documentation offers no reference for supported adapters or their required config fields. `build-and-run-odd-collectors.md` line 82 only links to a generic `config_examples/` tree in the old odd-collector repo. In reality the ecosystem has 60 adapters with distinct config requirements:

- `odd-collector/config_examples/` — 39 files
- `odd-collector-aws/config_examples/` — 11 files
- `odd-collector-gcp/config_examples/` — 3 files (plus 1 adapter without an example)
- `odd-collector-azure/config_examples/` — 4 files

Users have to find and read individual example YAMLs to discover what each adapter accepts. Configuration drift, missed fields, and "does this adapter exist?" questions are all inevitable without a reference.

## Acceptance Criteria

- [ ] An adapter-reference section exists under developer-guides
- [ ] One page per sub-collector (odd-collector, odd-collector-aws, odd-collector-gcp, odd-collector-azure)
- [ ] Each page lists all adapters in that collector with: adapter name, purpose, required config fields, optional config fields, defaults, and a working YAML example
- [ ] Every adapter that exists in code has an entry (coverage should match the count in each `config_examples/` directory, plus any adapters without example files)
- [ ] `build-and-run-odd-collectors.md` links into the adapter reference from the "configure collector-config.yaml" step
- [ ] Adapters without a `config_examples/` entry are flagged explicitly so the gap is visible for follow-up
