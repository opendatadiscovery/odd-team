## Description

The platform supports disabling (halting/snoozing) alert notifications per data entity, per alert type, for configurable durations (30 minutes to 1 week). Full UI (`AlertTypeRange.tsx`) and API (`GET/PUT /api/dataentities/{id}/alert_config`) exist, but docs don't mention the feature. Operators drowning in alerts for one flaky entity can't discover the snooze mechanism.

## Acceptance Criteria

- [ ] Alerting documentation includes a "halt notifications per entity" or "snooze" section
- [ ] The 4 alert types and 5 duration options are listed
- [ ] The `/api/dataentities/{id}/alert_config` endpoint is referenced
- [ ] A short UI walkthrough or screenshot shows where to configure halts
