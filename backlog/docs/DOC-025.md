## Rejection

**Closed as false positive on 2026-04-22.**

The Manual Object Tagging section on `origin/main` of the `documentation` repo (`docs/Features.md:173` area) has already been cleaned up — it uses "tags" only, with no "labels" terminology remaining. The stale-labels concern is resolved upstream.

**Root cause of the false finding**: the `docs/accuracy/feature-behavior` scanner ran against a stale branch (`hotfix/add_info_for_deployment_on_eks_to_summary`) that predated the terminology cleanup merged to `main` via the GitBook web editor.

**Note**: DOC-037 (permissions list referencing ghost `LABEL_*` permissions) is a **separate, still-valid** item — that's a platform-side permissions file, not the Features.md tagging section, and was not touched by the GitBook cleanup.

**Evidence**: `git -C ../documentation show origin/main:docs/Features.md` — Manual Object Tagging section contains no "labels" language.

## Description

Features.md lines 94-97 describe tags and labels as distinct concepts ("tags for entities, labels for fields"). Labels were eliminated in v0.17.0 (Oct 2023): the labels DB table was dropped and all labels auto-converted to tags. Docs were never updated. Users following the current docs will look for a labels UI/API that doesn't exist.

## Acceptance Criteria

- [ ] "Labels" is removed from the Manual Object Tagging section
- [ ] The section clarifies that tags apply to both entities and their fields (columns)
- [ ] No other doc page still describes labels as a separate concept (spot-check via search)
- [ ] Historical note is optional — the feature merge is visible in release notes, docs don't need to carry the history
