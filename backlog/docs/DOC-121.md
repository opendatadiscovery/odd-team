## Description

Editorial audit during `/review batch:feature/docs-editorial-audit-closeout` (2026-05-04) — page-level read of `docs/configuration-and-deployment/enable-security/authentication/oauth2-oidc.md` — surfaced a stray-character typo in the **Keycloak** YAML example.

Line 432 (Keycloak provider, `admin-principals` field):

```yaml
admin-principals: john,davidyam
```

Should be (matching every other admin-principals example on the page):

```yaml
admin-principals: john,david
```

Cross-references on the same page using the canonical `john,david` pattern:

- `auth.oauth2.client.cognito.admin-principals` example (line 82): `admin-principals: john,david`
- AWS Cognito env-var example (line 99): `AUTH_OAUTH2_CLIENT_COGNITO_ADMIN_PRINCIPALS=john,david`
- Github YAML example (line 136): `admin-principals: john,david`
- Github env-var example (line 156): `AUTH_OAUTH2_CLIENT_GITHUB_ADMIN_PRINCIPALS=john,david`
- Google YAML example (line 182): `admin-principals: john@odd.com,david@odd.com`
- Google env-var example (line 199): `AUTH_OAUTH2_CLIENT_GOOGLE_ADMIN_PRINCIPALS=john@odd.com,david@odd.com`
- Okta YAML example (line 394): `admin-principals: john@odd.com,david@odd.com`
- Okta env-var example (line 410): `AUTH_OAUTH2_CLIENT_OKTA_ADMIN_PRINCIPALS=john@odd.com,david@odd.com`
- Keycloak env-var example (line 448): `AUTH_OAUTH2_CLIENT_KEYCLOAK_ADMIN_PRINCIPALS=john,david`
- Keycloak with PKCE YAML example (line 476): `admin-principals: john,david`

The Keycloak YAML example at line 432 is the **only** example carrying `davidyam`. Almost certainly an editor finger-fumble (typing `david` and then accidentally hitting `y`, `a`, `m` on the next-row keys before pressing Enter) that survived review because the field is illustrative and the typo doesn't break a literal copy-paste — the operator reads it, mentally substitutes their own admin usernames, and never notices the doc said something other than `david`.

## Why this slipped past prior batches

The page covers 6+ OAuth providers across ~500 lines; per-provider examples are visually similar (each has its own YAML and env-var tab pair), so a reviewer scanning the page reads the structure rather than the exact field values. Spell-check on `davidyam` returns no match (it's a plausible-sounding made-up name) and code-block content is typically not subject to prose-level copy-edit. The mismatch only surfaces on a careful per-page cross-example consistency read — which is what the framework's axis A12 (Prose and rendering) audit cue calls for.

DOC-113 (cross-page A12 batch) does NOT cover this page. This finding is the only A12 defect surfaced on `oauth2-oidc.md` during the 2026-05-04 audit.

Editorial-audit case-law for `pillars/documentation/quality-framework.md` axis **A12** (Prose and rendering) — failure shape: **Prose polish** (stray-character typo in code-block content).
