## Description

The Azure AD section in `oauth2-oidc.md` (lines 204-360) covers config keys, prerequisites, single/multi-tenant setup, admin-attribute, admin-principals, admin-groups, troubleshooting — but never mentions `logout-uri`. The Cognito section documents `logout-uri` as a generic OAuth2 property at `oauth2-oidc.md:57`, but Azure operators reading only the Azure section do not learn that they need it.

The platform implements an Azure-specific logout flow that requires `logout-uri` to be set:

`AzureLogoutSuccessHandler.java:30-48`:

```java
public Mono<Void> handle(...) {
    final var uri = UriComponentsBuilder
        .fromUri(URI.create(provider.getLogoutUri()))   // ← NPE if logout-uri unset
        .queryParam("post_logout_redirect_uri", UriUtils.getBaseUri(requestUri))
        .queryParam("client_id", provider.getClientId())
        .encode(StandardCharsets.UTF_8)
        .build()
        .toUri();
    response.getHeaders().setLocation(uri);
    return exchange.getExchange().getSession().flatMap(WebSession::invalidate);
}
```

`provider.getLogoutUri()` returns null when the Azure config does not set `logout-uri`. `URI.create(null)` then throws NPE. Operators who configure Azure SSO without setting `logout-uri` will see logout fail in production.

For Azure AD, the canonical logout endpoint shape is `https://login.microsoftonline.com/{tenant-id-or-organizations-or-common}/oauth2/v2.0/logout`. This needs to be documented for both single-tenant and multi-tenant setups.
