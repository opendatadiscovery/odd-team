## Description

`oauth2-oidc.md` line 231, under the OKTA provider's env var example, sets `AUTH_OAUTH2_CLIENT_OKTA_PROVIDER=google`. Users who copy-paste this will configure OKTA-shaped auth with `provider=google`, which mismatches the client-id under the `okta.*` key and causes authentication failures. The YAML tab (line 215) correctly uses `provider: okta` — only the env-var tab has the bug, which is a classic copy-paste from the Google section above.

**Consumer-read**: the runtime-relevant consumer for this fix is `AzureUserHandler.shouldHandle()` (and its sibling `OktaUserHandler`, `GoogleUserHandler`, etc.), where `provider.equalsIgnoreCase(Provider.AZURE.name())` / `Provider.OKTA.name()` is the dispatch predicate. A mis-set `provider=google` under the `okta.*` key would route the request through `GoogleUserHandler` instead of `OktaUserHandler`, with downstream mismatches on `user-name-attribute` defaults. The fix ensures the dispatch resolves correctly.
