## Description

The metrics section of `odd-platform.md` documents only 2 of 4 metrics config keys defined in `application.yml` lines 158-163. Missing:

- `metrics.storage` — values `INTERNAL_POSTGRES` or `PROMETHEUS` (default: `INTERNAL_POSTGRES`)
- `metrics.prometheus-host` — the Prometheus URL when `storage=PROMETHEUS` (default: `http://localhost:9090`)

Documented today: `metrics.export.enabled` and `metrics.export.otlp-endpoint` only. Users choosing between internal storage and Prometheus storage have no guidance.

## Acceptance Criteria

- [ ] `odd-platform.md` metrics section documents `metrics.storage` with both valid values and the default
- [ ] `metrics.prometheus-host` is documented with its default and a note that it is only used when `storage=PROMETHEUS`
- [ ] Env-var equivalents are listed for both new keys
- [ ] A short paragraph explains the trade-off between internal storage (zero dependencies) and Prometheus (scales better for operational metrics)

## Review-ready notes (2026-04-24)

**What shipped** (commit `3e52e31` on `feature/docs-odd-platform-config`, `documentation` repo):

1. **Section restructured** into two subsections — "Metric storage backend" (`metrics.storage` + `metrics.prometheus-host`) and "Metric export to OTLP" (`metrics.export.enabled` + `metrics.export.otlp-endpoint`) — because they are independent concerns that share the `metrics.*` namespace.
2. **`metrics.storage`** documented with both valid values (`INTERNAL_POSTGRES` default, `PROMETHEUS`), what each backend does (UI read-back: internal Postgres tables vs Prometheus instant-query API), and when to pick which.
3. **`metrics.prometheus-host`** documented with default (`http://localhost:9090`) and the note that it is only consulted when `storage=PROMETHEUS`.
4. **Env-var equivalents** added for both new keys (`METRICS_STORAGE`, `METRICS_PROMETHEUS_HOST`).
5. **Trade-off paragraph** explains internal storage (zero dependencies) vs Prometheus storage (scales for operational metrics; already-running Prometheus).
6. **Warning admonition** (new, `{% hint style="warning" %}`) explicitly names the `IllegalStateException: Prometheus host is not defined` failure at startup when `metrics.storage=PROMETHEUS` with empty `metrics.prometheus-host`.
7. **Incidental fix** folded in: existing env-var block used `METRICS_EXPORT_OTLP-ENDPOINT={otlp-endpoint-url}` with a dash; Spring Boot env-var form uses underscores only, so the correct form is `METRICS_EXPORT_OTLP_ENDPOINT`. Noted in commit body.
