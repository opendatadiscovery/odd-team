## Description

Direct user request 2026-05-07: *"please log items to backlog to enrich `https://docs.opendatadiscovery.org/features/data-discovery` with dedicated features for data discovery — for instance, Search (full text search and facet search), tagging, DEG as a Domain, and all applicable others; also create a section for Data Lineage and for Data Glossary with dedicated topics — so the task is the same as for other aspect — leave on Features Overview only base information about a feature but mainly leave the references for dedicated pages with more details described. Still Management and Active platform features at the bottom."*

This is the same content-homing refactor that DOC-082 (3-drift IA refactor — Directory + GenAI + Build a custom collector) and DOC-131 (Active platform features pillar carve-out) executed for their respective surfaces. What's left on Features.md today is a mix of:

- Sections that **already** point at a dedicated page (Alerting, Notifications, Activity Feed, Data Collaboration, GenAI assistant, Query Examples, Relationships, Lookup Tables, Catalog Overview, Directory) — these are correctly homed.

## Context

- **Why ADR-class is NOT needed here.** Unlike DOC-138 (which restructured the SUMMARY *root*), this item only adds children to existing SUMMARY groups. The structural framework (`## Features` group with pillar children) was settled by DOC-138's accepted ADR; this is its routine extension. No new architectural decision; just content homing per Cornerstone 5.
- **Cornerstone alignment.** Cornerstone 1 ("a first-time reader grasps the platform by reading the Features.md TOC") survives — the teasers stay, only the deep-dive content moves out. Cornerstone 2 (hierarchy depth reflects conceptual depth) is reinforced — pillar landings group their dedicated sub-pages. Cornerstone 5 (one canonical home per content type) is the binding constraint that drives this item.
- **Precedent.** DOC-082 established the local-IA-refactor pattern; DOC-131 established the pillar-carve-out pattern; this item applies both at scale.
- **Sub-item ordering.** DOC-142/143/144 (Data Discovery enrichments) can ship in any order — no inter-dependencies. DOC-145 / DOC-146 / DOC-147 (new pillars) are independent of each other and of the Data Discovery sub-items. The umbrella DOC-141 closes (`done`) only after all 6 sub-items close.
- **What stays as a teaser on Features.md.** Sections that don't justify their own page: Metadata Storage, ML Experiments, Catalog Overview page (already linked to Data Discovery anchor), Directory (already linked to dedicated page), Dataset schema diff, Adding Business Names, Integrating Vector Store Metadata, Filters to Include / Exclude Objects, Data Entity Statuses, Alternative Secrets Backend, Integration Wizards, Data Entity Attachments, "Recommended" panel, Custom navigation links, Metadata stale, M2M tokens. These are 1-3 paragraph teasers each and don't deserve their own pages — premature decomposition would dilute Features.md's role as the platform's "one-page index of features".
