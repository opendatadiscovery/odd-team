## Description

The platform ships with a GenAI integration that proxies natural-language questions to an external AI endpoint. Surface:

- `GenAIController.java` — `POST /api/genai/ask`
- `GenAIService` + `GenAIProperties`
- Config (`application.yml` lines 17-20): `genai.enabled` (default: false), `genai.url`, `genai.request_timeout`

Zero documentation exists. Users don't know the feature is shippable, how to enable it, or what API contract the external AI endpoint must implement.

## Acceptance Criteria

- [ ] A new `genai.md` documentation page covers what the feature does (proxy natural-language questions to an external AI service)
- [ ] Configuration section documents `genai.enabled`, `genai.url`, `genai.request_timeout` with defaults and env-var equivalents
- [ ] The external AI endpoint API contract is specified (request/response shape expected by `GenAIService`)
- [ ] The `/api/genai/ask` endpoint is referenced
- [ ] UI entry point is described (how users access the "ask" affordance in the platform UI, if any)
- [ ] Features.md references the page
- [ ] SUMMARY.md includes the new page
- [ ] Platform config page links to the GenAI page from the relevant config section
