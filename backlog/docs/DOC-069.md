## Description

DOC-049 documented the `odd.links` configuration and the App Info menu rendering on `docs/configuration-and-deployment/odd-platform.md`, but `Features.md` (the user-facing feature index) does not mention the feature at all. A user browsing `Features.md` to discover what ODD does will not learn that admins can add custom links into the App Info menu.

The feature is small admin chrome — closer in scope to the existing "Recommended panel on the main page" paragraph at `Features.md:849` than to substantive features like Search / Lineage / Alerting — so it does not warrant a full section. A short teaser cross-linking to the canonical config-side documentation is enough to close the bidirectional code↔doc gap (Quality Bar Gate 6) without diluting the marketing-facing features around it.

## Acceptance Criteria

- [ ] A new short subsection (3–6 lines) titled "Custom navigation links" — or a single paragraph under a comparable existing umbrella — exists in `Features.md`, slotted near the "Recommended panel on the main page" cluster (currently `Features.md:849`)
- [ ] Content explains in user-facing language: admins can add a list of custom links (title + URL) that surface in the App Info menu (top-right toolbar, Information icon); typical use cases — internal wikis, runbooks, dashboards
- [ ] The teaser cross-links to the `odd.links` configuration section on `docs/configuration-and-deployment/odd-platform.md` for setup details (no parallel YAML/env-var examples on Features.md)
- [ ] The teaser is explicitly framed as admin customization (so a reader without admin access does not expect to be able to add links themselves)
