## Description

Carve out the **Data Entity Groups** content from `docs/Features.md` (L152-158) into a dedicated `docs/data-discovery/groups-domains.md` page that introduces both DEG-as-logical-grouping and DEG-as-Domain framings. The user named this explicitly (*"DEG as a Domain"*) — DEGs are the platform's logical-grouping primitive, and a DEG can be flagged as a **domain** so it surfaces in the Catalog Overview's Domains section (per the `catalog-overview.png` doc text added 2026-05-07 batch — *"Domains (conditional — appears when at least one Data Entity Group is flagged as a domain)"*). Today this Domain framing is mentioned only inline in the Catalog Overview description; it has no canonical home.

The page is **discovery-oriented**: how DEGs help find related entities (lineage, ownership, tag inheritance), the Domain flag as a top-level discovery surface, and the relationship to ML Experiments (which are DEGs of a particular shape — already documented at Features.md `## ML Experiments`).

Sub-task of the umbrella **DOC-141**.

## Context

- User wording: *"DEG as a Domain"* — the page should make the framing explicit (DEG is the primitive; Domain is a particular use of it). Avoid the temptation to introduce two separate pages.
- The Catalog Overview's Domains section is conditional today (only renders when domain-flagged DEGs exist). The new page is where an operator learns *how* to make Domains exist.
- The ML Experiments framing on Features.md L132-140 explicitly says *"In ODD, an ML experiment is a Data Entity Group …"* — that cross-link must update with this carve-out.

## Implementation Notes

- Effort medium — content enrichment beyond Features.md L152-158 is needed (the Domain framing and the operator workflow for flagging a DEG as a domain are not currently documented anywhere — consumer-read is required).
- Page filename `groups-domains.md` keeps both framings in the URL (operator searches for "groups" or "domains" should both land here). Alias considered: `domains.md` — but that loses the DEG-as-primitive framing in the URL.
