## Description

DOC-045 review (`/review` session, 2026-04-30) caught three small gaps that did not warrant blocking the merge but should be cleaned up in a follow-up commit. All three relate to the GenAI feature documentation that DOC-045 shipped.

### Gap 1 — Asymmetric HTTP status coverage in `docs/genai.md` (user-facing)

The "Platform endpoint" section (`docs/genai.md` ~lines 120-124) documents the disabled-feature error path as **"`BadUserRequestException` carrying the message 'Gen AI is disabled' (HTTP 400)"** — the HTTP status is explicit. The two GenAIException paths immediately below (timeout case, and non-timeout-error case) name the exception class but **omit the HTTP status code**. The actual status is **HTTP 500** for both, mapped in `ControllerAdvice.java:55-56` (`@ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)` on `handleGenAIException`).

An API integrator reading the page would have to test the endpoint to discover the status code, or guess (4xx is a common assumption for upstream-failure responses; the platform always returns 5xx for the GenAI error paths). Adding "(HTTP 500)" or equivalent to both paragraphs makes the section symmetric with the disabled case.

### Gap 2 — Missing bean factory in `navigation/domains/genai.md` (maintainer-internal)

CLAUDE.md "Token Budget Protocol" requires: *"Navigation domains must list bean factories and SDK builders, not just the controller/service/repository chain — consumer-read audits depend on them being findable without grepping."*

`navigation/domains/genai.md` lists Controller, Service, Configuration, and Properties — but **omits `WebClientConfiguration.java`**, which is the `@Bean("genAiWebClient")` factory and the WebClient/HttpClient builder for the outbound integration. DOC-045's Sources footer cites this file, but the navigation domain doesn't surface it for the next consumer-read audit.

### Gap 3 — Incorrect default in `navigation/domains/genai.md` line 18

Line 18 reads: `genai.request_timeout` — timeout in minutes (default: 2)`

The Java field default is `0` (primitive `int`); `2` is the value shown in `application.yml:20` as a **commented-out example** (`# request_timeout: 2`). The same file already documents the actual default correctly at lines 31-32 ("`genai.request_timeout` defaults to 0 (Java primitive default) → `Duration.ofMinutes(0)` immediate timeout"). The line-18 framing repeats the misframing class that DOC-045's user-facing page corrects — it should be aligned.
