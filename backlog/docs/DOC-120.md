## Description

Editorial audit during `/review batch:feature/docs-editorial-audit-closeout` (2026-05-04) — page-level read of `docs/configuration-and-deployment/enable-security/authorization/policies.md` — surfaced **seven broken intra-page anchors** in a single section.

### Site A — line 59 (two broken anchors in one sentence)

```markdown
In ODD Platform we have pre-defined [condition operators](policies.md#condition-operations) and [fields](policies.md#con), which can be used with these operators.
```

Two defects:

1. `policies.md#condition-operations` — typo for `policies.md#condition-operators`. The H4 heading on line 61 reads `#### Condition operators`; GitBook's slugger generates `#condition-operators`. The link's `#condition-operations` is a wrong-noun typo (operations vs operators).
2. `policies.md#con` — truncated for `policies.md#condition-fields`. The H4 heading on line 74 reads `#### **Condition fields**`; GitBook's slugger generates `#condition-fields`.

### Site B — lines 67-72 (six broken anchors in one bullet list)

The operator bullet list under `#### Condition operators`:

```markdown
* `eq` \*\*\*\* - [condition field](policies.md#con) must be equal to some value
* `not_eq` \*\*\*\* - \*\*\*\* [condition field](policies.md#con) must not be equal to some value
* `match` \*\*\*\* - [condition field](policies.md#con) must match some value
* `not_match` \*\*\*\* - \*\*\*\* [condition field](policies.md#con) must not match some value
* `is` \*\*\*\* - \*\*\*\* [condition field](policies.md#con) must be true
* `not_is` \*\*\*\* - \*\*\*\* [condition field](policies.md#con) must be false
```

Each bullet's `(policies.md#con)` is a truncated reference for `(policies.md#condition-fields)`. All six fail to resolve on the live site.

(The `\*\*\*\*` escaped-asterisk rendering artefact on the same lines is a separate A12 defect already catalogued under DOC-113. This finding is scoped to the broken anchors only.)

### Live-site verification (already performed)

WebFetch `https://docs.opendatadiscovery.org/configuration-and-deployment/enable-security/authorization/policies` (during the audit, 2026-05-04) confirmed:

- The "Condition operators" section renders, but the bullets carry literal `****` artefacts.
- The `#con` anchor does not exist on the page.
- The `#condition-fields` anchor exists and corresponds to the `#### **Condition fields**` heading on line 74.
- The `#condition-operators` anchor exists and corresponds to the `#### Condition operators` heading on line 61.
- A reader clicking any of the seven broken links lands at the page top with no scroll — silent failure.

## Why this slipped past prior batches

This is one of those defects that is **invisible at authoring time**: the link text reads "condition field" / "condition operators" so the rendered prose looks coherent; the broken anchor only manifests on click-through. The doc-product editorial audit's per-page A4 cue (`for each [text](target) on the page: does the target exist? Does the anchor resolve?`) is exactly the check that catches this — without that audit cue applied per-link, the rendering looks acceptable.

DOC-113 (cross-page A12 batch) catalogues the `\*\*\*\*` escaped-asterisk artefacts on these same lines (67-72) but does **not** cover the broken anchors — DOC-113's scope is A12 (rendering), this finding is A4 (reference integrity).

Editorial-audit case-law for `pillars/documentation/quality-framework.md` axis **A4** (Reference integrity) — failure shape: **Dead anchor** (the in-page link target does not exist; reader gets silent no-op).
