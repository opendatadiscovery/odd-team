## What

The `OdbcPlugin.driver` field in the generic `odd-collector` package declares its default as `"{ODBC Driver 17s for SQL Server}"`. The `s` after `17` is a typo — there is no SQL Server ODBC driver registered with that name on any standard distribution. Operators relying on the model default — i.e., omitting `driver:` from their `collector_config.yaml` — will see the adapter fail at connection time because no ODBC driver matches the typoed string.

The reference YAML at `odd-collector/config_examples/odbc.yaml` uses the **correct** value `"{ODBC Driver 17 for SQL Server}"` (no `s`), so operators who copy from that file are fine. Operators who don't supply a `driver:` value at all are broken.

## Where

`odd-collector/odd_collector/domain/plugin.py:37-41`:

```python
class OdbcPlugin(DatabasePlugin):
    type: Literal["odbc"]
    driver: str = "{ODBC Driver 17s for SQL Server}"  # <-- typo: "17s" should be "17"
    password: Optional[SecretStr]
```

Compare with `odd-collector/config_examples/odbc.yaml:11`:

```yaml
    driver: "{ODBC Driver 17 for SQL Server}"
```

## Why it matters

- An operator who reads only the field name and assumes "it has a default — I can omit it" gets a runtime error from `pyodbc` because no driver named `{ODBC Driver 17s for SQL Server}` is registered on their container.
- The error message points at the ODBC layer, not the plugin's default value, making the root cause non-obvious. Time-to-diagnose is high (operators have to read the Pydantic source to find the typo).
- Severity is **medium** — operators who follow the reference YAML are unaffected, but the field-level Pydantic default is the surface most operators trust without copying examples.

## Suggested fix

Replace the model default with the correct driver name (`{ODBC Driver 17 for SQL Server}`) — or, more conservatively, drop the default entirely and require operators to supply `driver:` explicitly (since the right driver depends on what's installed on their container). Either change is one line.

```python
class OdbcPlugin(DatabasePlugin):
    type: Literal["odbc"]
    driver: str = "{ODBC Driver 17 for SQL Server}"  # was: "17s"
    password: Optional[SecretStr]
```
