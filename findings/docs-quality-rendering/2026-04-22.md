### F-R03: SUMMARY.md escapes underscores inconsistently

GitBook tolerates this â€” the live links resolve â€” but it prevents any simple grep-based tool (including this scanner) from diffing SUMMARY.md against the filesystem without first normalizing.
- **Live URL**: each affected page renders correctly; no user-visible bug today
- **Severity**: **Medium** â€” no live breakage; tooling hazard. This is why step 1 of every future rendering run needs an explicit unescape pass before comparing SUMMARY entries to files.
- **Suggested fix**: normalize to plain underscores in a future GitBook-backed edit (carefully â€” hand-edits to SUMMARY.md have historically caused cache issues; see 2026-04-22 S2S incident). Lower priority than the above findings. Meanwhile, document the escaping in the scanner method as "always unescape before diffing."
