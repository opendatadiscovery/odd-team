## Description

Editorial audit during `/review batch:feature/docs-editorial-audit-closeout` (2026-05-04) — page-level read of `docs/developer-guides/build-and-run/README.md` — surfaced the page as a **sparse landing-page stub**.

Full content of the file (7 lines):

```markdown
---
description: Build and run section of the whole ODD infrastructure
---

# Build and run

This article shows how to build and run code for each part of ODD infrastructure, including ODD Platform backend and frontend and ODD Collectors
```

The page lists no sub-pages. A reader hitting this page expects to be told where to go for the platform and the collector; instead they get a one-sentence intro and an invisible exit. SUMMARY.md does carry the sub-pages so the reader can navigate via the sidebar — but the body of the section landing should provide the same affordance.

For comparison, the sibling **enable-security** subtree carries proper section landings:

- `enable-security/authentication/README.md` — has a `### Contents` block with a one-line description per sub-page (`Disable authentication`, `Login form`, `OAUTH2/OIDC`, `LDAP`, `S2S`).
- `enable-security/authorization/README.md` — same pattern (`Policies`, `Permissions`, `Roles`, `Owners`, `User-owner association`).

The build-and-run section should follow the same pattern.

## Why this slipped past prior batches

The page has been a one-line stub since early ODD docs era. Section-landing-page completeness was not part of any prior audit's scope until the framework codification (`pillars/documentation/quality-framework.md`, 2026-05-03) introduced axis **A7** (Reader-flow integrity) with the audit cue "Walk the page as a first-time reader of the topic. At each step-by-step, do all the steps exist?" — a section landing with no contents block fails this cue when the section has multiple sub-pages the reader is expected to navigate among.

Editorial-audit case-law for `pillars/documentation/quality-framework.md` axis **A7** (Reader-flow integrity) — failure shape: **Sparse landing page** (section landing with no contents/index block; readers depend on sidebar navigation rather than in-body cross-links).
